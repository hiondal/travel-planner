# 유저스토리 리뷰 — 마법사 (AI 엔지니어)

> 리뷰 관점: AI/ML 적용 가능성, AI 기능 누락 여부, AI 연동 요구사항
> 리뷰일: 2026-02-22

---

## 긍정 평가

1. **분석 이벤트 4종 수집 구조 확보 (P18)**
   UFR-MNTR-040, UFR-BRIF-050, UFR-ALTN-020, UFR-ALTN-030에 배지상세조회됨 / 브리핑열람됨 / 대안카드노출됨 / 대안카드채택됨 이벤트가 명시적으로 포함되었다. 이 4개 이벤트는 향후 AI 추천 모델의 학습 데이터셋 핵심 라벨이 될 수 있는 행동 시그널이다. 마법사가 A3에서 요청한 사용자 반응 피드백 루프의 최소 기반이 갖춰진 것을 긍정적으로 평가한다.

2. **대안 미채택 사유 수집 (UFR-ALTN-040)**
   닫기/시간초과 사유를 기록하고 "미채택 데이터가 추천 품질 개선에 활용된다"는 체크리스트 항목이 명시되었다. A4 요청 사항이 의도에 맞게 반영되었다. 비선택 데이터는 선택 데이터보다 AI 학습에서 더 많이 수집되는 경향이 있어 데이터 불균형 해소에도 기여한다.

3. **수집 데이터 품질 메타데이터 포함 (UFR-MNTR-010, 시나리오 2)**
   is_cached=true, data_age_seconds, source_status=FALLBACK 메타데이터가 수집 실패 시 응답에 포함된다. A2 요청의 핵심인 데이터 신뢰도 추적 기반이 갖춰졌다. 향후 Confidence Score 계산 시 이 메타데이터가 입력 피처로 활용될 수 있다.

4. **MoSCoW Won't 항목에 AI 자동 일정 재조정 명시**
   섹션 6-1의 Won't 항목에 "AI 자동 일정 재조정"이 명시적으로 위치한다. MVP 범위를 명확히 선을 그은 것은 올바른 판단이다. 단, 이 항목이 Phase 3 로드맵과 연결되어 있으므로 현 설계에서 해당 확장을 막지 않는 구조가 되어야 한다.

5. **대안 카드 선택 순위 기록 (UFR-ALTN-030)**
   채택된 카드의 선택 순위(1/2/3번)가 기록된다. 단순 채택 여부보다 정렬 알고리즘의 실효성을 검증하는 데 필수적인 데이터다. 현재 Rule-based 정렬(거리 > 평점 > 혼잡도)의 유효성을 통계적으로 검증하고, 이후 AI 기반 학습 랭킹 모델로 전환할 때 Ground Truth로 사용할 수 있다.

## AI 적용 가능 UFR 분석

### UFR-MNTR-020: 상태 배지 판정

| 항목 | 내용 |
|------|------|
| **현재 방식** | Rule-based. 초록/노랑/빨강/회색 4단계를 고정 임계값으로 판정. 영업 중 + 날씨 양호 + 혼잡도 보통 이하 + 이동시간 정상 = 초록 |
| **AI 고도화 가능성** | 높음. 4개 시그널(영업상태, 날씨, 혼잡도, 이동시간)을 피처로 하는 다변량 분류 모델로 전환 가능. 특히 "혼잡도 보통 이하"와 같이 임계값을 사람이 고정하는 부분에서 사용자 행동 기반 자동 임계값 학습이 가능하다. |
| **핵심 공백** | Confidence Score(A1)가 UFR에 없다. 상태 판정 결과와 함께 신뢰도 수치를 저장하는 필드가 현재 체크리스트에 없다. Rule-based 시대에도 데이터 품질(is_cached, data_age_seconds)을 기반으로 낮은 신뢰도 판정 결과를 조기에 식별할 수 있어야 한다. |
| **전환 조건** | 동일 장소에 대한 판정 데이터 1,000건 이상 누적 시 AI 분류 모델 학습 가능. 현재 UFR-ALTN-030의 선택 순위 데이터와 조합하면 "정확한 판정"의 Ground Truth 생성 가능. |

### UFR-ALTN-010: 대안 장소 검색 및 정렬

| 항목 | 내용 |
|------|------|
| **현재 방식** | Rule-based 정렬. 1순위 거리(가까운 순), 2순위 평점(높은 순), 3순위 혼잡도(낮은 순) 고정 가중치. |
| **AI 고도화 가능성** | 매우 높음. 사용자의 선택 이력(선택 순위 데이터)을 기반으로 개인화된 정렬 가중치 학습이 가능하다. 예: 빠른 이동을 선호하는 사용자에게는 거리 가중치 강화, 음식 퀄리티를 중시하는 사용자에게는 평점 가중치 강화. |
| **핵심 공백** | 현재 정렬 기준이 고정 3단계 우선순위로 되어 있어 AI 전환 시 인터페이스 변경이 필요하다. 정렬 함수를 점수 기반 합산(score = w1*거리점수 + w2*평점 + w3*혼잡도점수)으로 추상화하면 Rule-based와 AI 모델 간 전환이 코드 변경 없이 가능하다. 이 구조 변경이 유저스토리에 언급되지 않았다. |
| **전환 조건** | UFR-ALTN-030의 선택 순위 + UFR-ALTN-040의 미채택 사유 데이터 500건 이상 시 Learning-to-Rank 모델 학습 가능. |

### UFR-BRIF-010: 브리핑 생성 (텍스트 총평)

| 항목 | 내용 |
|------|------|
| **현재 방식** | 구조화 데이터 기반 템플릿 출력. "모두 정상입니다"(안심), "임시 휴업", "폭우 예보"(주의) 등 고정 텍스트. |
| **AI 고도화 가능성** | 높음. 영업상태, 날씨, 혼잡도, 이동시간 수치를 입력받아 자연어 요약 생성. A6에서 제안한 브리핑 텍스트 생성 단계 분리가 Phase 3에 배정되어 있으나, MVP에서도 총평 필드의 생성 방식을 "외부 AI 호출 가능한 인터페이스"로 추상화해두면 Phase 3 전환 비용이 크게 감소한다. |
| **핵심 공백** | UFR-BRIF-010 체크리스트에 총평(summary) 생성 방식이 명시되지 않았다. Rule-based 텍스트 조합인지, AI LLM 생성인지 불명확하다. NFR에도 AI 응답 지연에 대한 처리 전략이 없다. |
| **전환 조건** | LLM 호출 비용 및 응답 지연이 NFR-PERF-010(p95 2초 이내)을 위반할 수 있다. 비동기 생성 + 캐싱 전략이 Phase 3 설계 시 필수다. |

### UFR-MNTR-010: 외부 데이터 수집 (데이터 품질)

| 항목 | 내용 |
|------|------|
| **현재 방식** | 15분 주기 수집, 실패 시 캐시 fallback. is_cached, data_age_seconds 메타데이터 포함. |
| **AI 고도화 가능성** | 중간. 수집 데이터 품질 지표(is_cached 비율, data_age_seconds 분포)를 모니터링하면 특정 도시/시간대의 데이터 신뢰도를 자동 평가할 수 있다. 신뢰도가 낮은 시간대에는 수집 주기를 단축하는 적응형 수집 전략도 가능하다. |
| **핵심 공백** | 현재 수집 성공/실패 여부만 기록되고, 수집된 데이터의 이상값(anomaly) 탐지 로직이 없다. 예: 혼잡도가 이전 값 대비 100% 급등한 경우 API 오류인지 실제 혼잡도 급증인지 판별하는 로직이 없다. |

### UFR-ALTN-030: 대안 카드 선택 (피드백 루프 완결성)

| 항목 | 내용 |
|------|------|
| **현재 방식** | 선택 완료 후 분석 이벤트(채택됨, 선택 순위) 기록. |
| **AI 고도화 가능성** | 높음. 선택 후 실제 여행자 만족도(장소 방문 후 재방문/이탈 여부)까지 추적하면 "선택한 대안의 실제 품질"을 학습할 수 있다. 단, 이 추가 신호는 현재 UFR 범위 밖이다. |
| **핵심 공백** | 선택 순위 기록은 있으나, 선택된 대안의 정렬 점수(거리점수, 평점, 혼잡도 수치) 자체가 기록되지 않는다. 나중에 "왜 1번 카드를 선택했는가"를 분석하려면 카드 제시 시점의 각 점수 스냅샷이 필요하다. |

## 개선 의견

### 개선 1: Confidence Score 필드 추가 (UFR-MNTR-020)

**현재 상태:** 판정 결과(초록/노랑/빨강/회색)와 판정 사유만 저장.

**보완 제안:** 판정 결과와 함께 `confidence_score` (0.0~1.0) 필드를 저장하는 체크리스트 항목을 추가한다.

```
[체크리스트 추가 항목]
- 판정 결과와 함께 신뢰도 점수(confidence_score)가 저장된다
  - 신뢰도 계산 기준: 캐시 데이터 비율, data_age_seconds 평균, 수집 성공률
  - confidence_score 0.5 미만 시 회색 배지와 동일하게 "데이터 불확실" 표시 여부 검토
```

MVP에서는 Rule-based로 계산하되 (캐시 없음 = 1.0, 일부 캐시 = 0.7, 전체 캐시 = 0.4), Phase 2에서 ML 기반으로 전환하는 경로가 명확해진다.

---

### 개선 2: 대안 정렬 인터페이스 추상화 (UFR-ALTN-010)

**현재 상태:** "1순위 거리(가까운 순), 2순위 평점(높은 순), 3순위 혼잡도(낮은 순)" 고정.

**보완 제안:** 정렬 로직을 합산 점수 방식으로 정의하는 NFR 또는 설계 주석 추가.

```
[설계 주석 추가]
- 대안 카드 정렬은 score = w1*distance_score + w2*rating_score + w3*congestion_score
  합산 점수 기반으로 구현한다
- MVP 가중치: w1=0.5, w2=0.3, w3=0.2 (거리 우선 Rule-based)
- Phase 2+: 사용자 선택 이력 기반 가중치 자동 학습으로 전환
```

이 인터페이스 추상화가 유저스토리에 명시되지 않으면 개발 단계에서 고정 if-else 정렬로 구현될 가능성이 높고, 이후 AI 전환 시 대규모 리팩토링이 필요해진다.

---

### 개선 3: 대안 카드 노출 시 점수 스냅샷 저장 (UFR-ALTN-020)

**현재 상태:** 분석 이벤트(대안 카드 노출됨)에 노출 카드 수, 검색 반경만 기록.

**보완 제안:** 카드 노출 시점의 각 후보 점수(거리, 평점, 혼잡도 수치)를 함께 저장한다.

```
[체크리스트 추가 항목]
- 분석 이벤트(대안 카드 노출됨)에 각 카드의 거리(m), 평점(0.0~5.0),
  혼잡도(낮음/보통/혼잡 수치) 스냅샷이 포함된다
```

이 데이터가 없으면 "왜 1번 카드가 선택되었는가"에 대한 Counterfactual 분석이 불가능하다. Learning-to-Rank 모델 학습의 필수 피처다.

---

### 개선 4: 브리핑 총평 생성 방식 명시 (UFR-BRIF-010)

**현재 상태:** 총평 필드가 체크리스트에 포함되어 있으나 생성 방식이 불명확.

**보완 제안:** 시나리오 설명 또는 체크리스트에 총평 생성 방식을 명시한다.

```
[체크리스트 수정]
- 총평은 수집 데이터 기반 템플릿 텍스트로 생성된다 (MVP)
  예: 안심 - "현재까지 모든 항목 정상입니다. 예정대로 출발하세요."
      주의 - "{위험항목} 이(가) 감지되었습니다. 대안을 확인해보세요."
- 총평 생성 로직은 외부 호출 가능한 인터페이스로 추상화된다 (Phase 3 AI 전환 대비)
```

---

### 개선 5: 알림 수신-열람 간격 활용 명시 (UFR-BRIF-050)

**현재 상태:** "알림 수신-열람 간격(초)이 분석 데이터로 기록된다"는 체크리스트가 있으나 활용 목적이 불명확.

**보완 제안:** 수집 목적을 명시하여 데이터 중요성을 팀 내에 공유한다.

```
[체크리스트 주석 추가]
- 알림 수신-열람 간격이 분석 데이터로 기록된다 (P18)
  활용 목적: Push 알림 적정 발송 시간 최적화, 브리핑 트리거 타이밍(15~30분 전) 효과 검증
```

## 누락 사항

### 누락 1: Confidence Score 저장 UFR 부재 (A1 미반영)

리뷰반영결과에서 A1(상태 판정 신뢰도 점수)은 "AI 모델 고도화 시" 반영으로 분류되어 있다. 그러나 Confidence Score를 나중에 추가하려면 스키마 변경과 기존 데이터 소급이 필요하다. MVP부터 `confidence_score` 필드를 NULL 허용으로 추가해두면 AI 전환 비용이 대폭 감소한다. 현재 유저스토리 어디에도 이 필드가 언급되지 않는다.

**제안:** NFR 또는 UFR-MNTR-020 체크리스트에 "판정 결과와 함께 confidence_score 필드를 NULL로 초기화하여 저장한다 (향후 AI 모델 입력을 위한 컬럼 예약)" 문구를 추가한다.

---

### 누락 2: 여행 스타일 프로파일 수집 기반 없음 (A5 미반영)

A5(여행 스타일 프로파일 이벤트)는 Phase 2+ 개인화 대상이다. 그러나 개인화를 위해서는 MVP부터 최소한의 사용자 행동 데이터를 축적해야 한다. 현재 분석 이벤트 4종(P18)만으로는 여행 스타일 프로파일을 구성하기에 신호가 부족하다.

MVP에서 추가 수집이 필요한 묵시적 시그널:
- 브리핑 열람 후 "대안 보기" 진입 비율 (위험 회피 성향 지표)
- 대안 선택 시 선택된 카드의 특성 (거리 최소/평점 최대 중 어느 쪽 선택)
- 반경 확장 발생 빈도 (해당 사용자가 좁은 반경을 선호하는지)

이 시그널들은 현재 P18 이벤트에 포함되지 않거나 불완전하게 포함된다. "미채택 사유(닫기/시간초과)"만으로는 거절 이유를 파악하기 어렵다.

**제안:** UFR-ALTN-040 체크리스트에 "카드별 거주 시간(노출 후 탭 없이 화면에 머문 시간)을 기록한다" 항목을 추가한다.

---

### 누락 3: StatusHistory 보존 정책 부재 (A7 미반영)

A7(StatusHistory 보존 정책)은 "데이터 설계 단계"로 분류되어 있으나, 보존 기간 및 집계 정책이 결정되지 않으면 AI 학습 데이터 가용 범위가 불명확해진다.

UFR-MNTR-020 시나리오 1에 "판정 결과와 판정 사유가 함께 저장된다"는 항목이 있으나, 저장 보존 기간과 이력 관리 방식(덮어쓰기 vs. 이력 누적)이 명시되지 않았다.

**제안:** NFR 또는 UFR-MNTR-020에 "상태 판정 이력은 이력 누적 방식으로 저장되며, 최소 6개월 이상 보존한다 (AI 학습 데이터셋 구성 목적)"를 명시한다.

---

### 누락 4: AI 응답 지연에 대한 NFR 부재

현재 NFR-PERF-010(사용자 대면 API p95 2초)과 NFR-PERF-020(대안 검색 3초)이 있다. 그러나 Phase 3에서 LLM 기반 브리핑 텍스트 생성이 도입되면 이 NFR 기준과 충돌한다. LLM 호출은 통상 1~5초 소요되어 p95 2초 기준을 위반할 가능성이 높다.

**제안:** NFR 섹션에 "AI 생성 콘텐츠(브리핑 총평 등)는 비동기 처리 또는 사전 생성 캐싱으로 응답 지연을 방지한다"는 설계 제약을 선제적으로 추가한다.

---

### 누락 5: 데이터 가명처리와 AI 학습 활용 관계 불명확 (NFR-SEC-020)

NFR-SEC-020에 "분석 데이터는 가명처리 후 활용"이 명시되어 있다. 그러나 AI 모델 학습 시 개인화(사용자별 가중치 학습)와 가명처리는 상충 관계가 있다. 개인화 모델은 사용자 ID와 행동 이력의 연결이 필요하기 때문이다.

**제안:** NFR-SEC-020 체크리스트에 "AI 개인화 학습을 위한 행동 이력은 가명화된 사용자 ID를 기준으로 연결하며, 원본 사용자 ID와의 매핑 테이블은 별도 분리 보관한다"는 항목을 추가한다.

## MVP vs 고도화 로드맵 제안

### 전환 경로 개요

```
MVP (Rule-based)          Phase 2 (데이터 기반)     Phase 3 (AI 기반)
─────────────────────     ─────────────────────     ─────────────────────
상태 판정: 고정 임계값  →  임계값 통계 최적화      →  ML 분류 모델 (4-class)
대안 정렬: 거리>평점>   →  정렬 가중치 A/B 테스트  →  Learning-to-Rank 모델
          혼잡도 고정                                  (개인화 가중치)
브리핑 총평: 템플릿     →  템플릿 A/B 테스트       →  LLM 기반 자연어 생성
            텍스트         (클릭률 기반 선택)          (Claude API / GPT-4)
데이터 품질: 수동 메타  →  품질 점수 자동 계산     →  이상값 탐지 모델
            데이터만        (A2 기반)
개인화: 없음            →  여행 스타일 클러스터링  →  사용자별 추천 모델
                           (A5 기반)
```

---

### Phase별 상세 계획

#### MVP: Rule-based 기반 확립 (현재 Sprint 1~3)

- 상태 판정: 4단계 고정 임계값 Rule-based
- 대안 정렬: 거리 > 평점 > 혼잡도 고정 가중치
- 브리핑 총평: 템플릿 텍스트 조합
- **AI 전환 준비 작업 (추가 필요):**
  - PlaceStatus 테이블에 `confidence_score` 컬럼 NULL로 예약
  - Alternative 정렬을 합산 점수 인터페이스로 추상화
  - 분석 이벤트 4종 외 카드별 점수 스냅샷 저장
  - StatusHistory 이력 누적 저장 구조 확보

#### Phase 2: 통계 기반 고도화 (MVP 출시 후 3~6개월)

- 트리거 조건: 분석 이벤트 누적 3,000건 이상
- 주요 작업:
  - 대안 정렬 가중치 통계 최적화 (선택 순위 데이터 기반)
  - 상태 판정 임계값 통계 최적화 (배지 상세 조회 후 대안 진입 비율 분석)
  - 여행 스타일 클러스터링 (A5) 시작
  - Confidence Score Rule-based 계산 시작
- A5(여행 스타일 프로파일), A7(StatusHistory 보존) 반영

#### Phase 3: AI/LLM 기반 전환 (Phase 2 후 6~12개월)

- 트리거 조건: 선택 이력 10,000건 이상, LLM 비용 ROI 검증 완료
- 주요 작업:
  - 대안 정렬 Learning-to-Rank 모델 도입 (개인화)
  - 상태 판정 ML 분류 모델 도입 (Confidence Score 포함)
  - 브리핑 총평 LLM 생성 도입 (A6) — 비동기 처리 필수
  - AI 자동 일정 재조정 서비스(S03) 검토
- 이벤트 스토밍 Phase 3 "AI 재조정 서비스 추가" 조건 충족

## 총평

도그냥(서비스 기획자)이 작성한 유저스토리 초안은 이벤트 스토밍 22개 정책(P1~P22)을 빠짐없이 UFR에 매핑한 점, 분석 이벤트 4종(P18)과 미채택 데이터(C2) 수집을 명시한 점에서 AI 관점의 데이터 기반 설계 의도가 잘 반영되어 있다.

다만 마법사 관점에서 가장 우려되는 지점은 **"AI 전환을 위한 준비 코드와 데이터 구조가 MVP에 포함되지 않으면, Phase 2 이후 전환 비용이 Phase 1 재개발 수준이 될 수 있다"** 는 점이다. 특히 세 가지가 반드시 MVP에 포함되어야 한다.

1. `confidence_score` 컬럼 예약 (UFR-MNTR-020)
2. 대안 정렬 합산 점수 인터페이스 추상화 (UFR-ALTN-010)
3. StatusHistory 이력 누적 저장 방식 명시 (UFR-MNTR-020, A7)

이 세 가지는 사용자에게 보이지 않는 내부 설계 결정이기 때문에 유저스토리에서 생략되기 쉽다. 그러나 실용주의 AI 엔지니어 입장에서 "AI를 위한 AI"가 아닌 "실제 문제 해결을 위한 AI"로 가려면, MVP부터 올바른 데이터 구조와 인터페이스를 잡아두는 것이 가장 효율적인 경로다.

전체 유저스토리 품질은 22개 UFR + 10개 NFR + 전체 스프린트 계획이 정합성 있게 구성된 수준 높은 초안으로 평가한다. 위의 개선 의견 5개와 누락 사항 5개를 보완하면 AI 고도화 관점에서도 충분히 견고한 요구사항이 될 것이다.
