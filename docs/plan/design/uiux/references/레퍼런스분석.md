## 2. 레이아웃 패턴

### 홈 피드 레이아웃
- **스크롤 가능한 카드 리스트**: 세로 스크롤 기반, 전체 너비 카드
- **이미지/비디오 우선 카드**: 카드 상단에 미디어, 하단에 텍스트 오버레이
- **섹션 헤더**: 좌측 정렬, 간결한 타이틀 + 우측 "더보기" 링크

### 카드 레이아웃
- **풀 블리드 미디어 카드**: 이미지가 카드 전체를 덮는 형태
- **라운드 코너**: 모서리 반경 약 `12px` ~ `16px`
- **오버레이 텍스트**: 이미지 위에 그라디언트 오버레이 후 텍스트 배치
- **썸네일 그리드**: 2열 또는 3열 그리드로 콘텐츠 나열

### 바텀시트 (Bottom Sheet) 패턴
- **핸들 인디케이터**: 상단 중앙에 짧은 회색 바 (`width: ~36px, height: ~4px`)
- **배경**: 다크 그레이 (`#1A1A1A` ~ `#222222`)
- **확장 가능**: 절반 높이 → 전체 높이 드래그 지원
- **타이틀 영역**: 좌측 정렬 타이틀 + 우측 닫기(X) 버튼

### 그리드 시스템
- **콘텐츠 패딩**: 좌우 `16px`
- **카드 간격**: `8px` ~ `12px`
- **섹션 간격**: `24px` ~ `32px`

## 3. 타이포그래피

### 폰트 스타일
- **기본 폰트**: 시스템 산세리프 (iOS SF Pro 계열로 추정)
- **로고/브랜드명**: 볼드 산세리프, 대문자 혼용 ("WWIT - 윗", "Dot SlashDash")
- **한국어 지원**: 한/영 혼용 인터페이스

### 크기 계층
| 레벨 | 크기 추정 | 용도 |
|------|-----------|------|
| Display | `28px` ~ `32px` | 온보딩 메인 타이틀 |
| Heading 1 | `22px` ~ `24px` | 섹션 타이틀, 앱명 |
| Heading 2 | `18px` ~ `20px` | 카드 타이틀, 바텀시트 헤더 |
| Body | `14px` ~ `16px` | 본문 텍스트, 설명 |
| Caption | `11px` ~ `12px` | 날짜, 메타 정보, 서브 레이블 |

### 굵기 패턴
- **Bold (`700`)**: 앱명, 섹션 타이틀, CTA 버튼 텍스트
- **SemiBold (`600`)**: 카드 타이틀, 탭 레이블
- **Regular (`400`)**: 본문, 설명 텍스트
- **Light (`300`)**: 날짜, 보조 정보

### 특징
- "추억과 리워드" 같은 감성적 한국어 카피 사용
- 영문 브랜드명(DotSlashDash)과 한국어 UI 혼용
- 대문자 영문 강조: "REWARD SYSTEM OPEN"

## 4. 컴포넌트 스타일

### 버튼
| 유형 | 스타일 | 특징 |
|------|--------|------|
| 프라이머리 CTA | 레드 배경 + 화이트 텍스트 | 소문자+대문자 혼용, 라운드 코너 `8px` |
| 세컨더리 | 다크 그레이 배경 + 화이트 텍스트 | 아웃라인 없음, 플랫 스타일 |
| 텍스트 버튼 | 배경 없음 + 화이트/그레이 텍스트 | 언더라인 없음 |
| 토글 버튼 | 다크 배경, 활성 시 화이트 | 라운드 필 스타일 |

### 입력 필드
- 배경: `#2A2A2A` ~ `#333333` (다크 배경)
- 텍스트: 화이트
- 플레이스홀더: 회색 (`#666`)
- 보더: 없거나 매우 미세한 다크 보더
- 라운드 코너: `8px` ~ `12px`
- 좌측 아이콘 (검색 등) 포함 패턴 多

### 카드
- **미디어 카드**: 풀 블리드 이미지 + 하단 그라디언트 오버레이 + 텍스트
- **리스트 카드**: 좌측 썸네일(정사각/라운드) + 우측 텍스트 정보
- **컬렉션 카드**: 제목 + 서브타이틀 + 미디어 프리뷰 그리드
- 라운드 코너: `12px` ~ `16px`
- 그림자: 없음 (플랫 다크 배경에 레이어 분리)

### 네비게이션 바 (하단)
- 배경: 최상위 배경과 동일하거나 약간 밝은 다크
- 아이콘: 라인 아이콘 스타일 (비활성: 그레이, 활성: 화이트)
- 탭 레이블: 아이콘 하단에 소형 텍스트
- 구성: 홈 / 검색 / 업로드(+) / 마이페이지 / 리워드
- 세이프 에어리어(홈 인디케이터) 위에 위치

### 탭/세그먼트 컨트롤
- 배경: 다크 그레이 컨테이너
- 활성 탭: 화이트 배경의 필 스타일
- 비활성 탭: 투명 배경 + 그레이 텍스트
- 라운드 코너: `20px` ~ `24px` (완전한 필 형태)

### 토스트 / 알림
- 다크 배경 + 화이트 텍스트
- 하단 중앙 또는 상단에 표시
- 간결한 텍스트, 아이콘 없거나 소형 아이콘

### 팝업 / 모달
- 반투명 다크 오버레이 배경
- 중앙 정렬 카드 형태
- 라운드 코너 `16px` ~ `20px`
- 제목 + 설명 + 버튼 구성

### 바텀시트
- 핸들 바: 상단 중앙, `#444` ~ `#555`, `width 36px / height 4px`
- 배경: `#1C1C1C` ~ `#222222`
- 라운드 상단 코너: `20px` ~ `24px`
- 내부 콘텐츠: 리스트, 체크박스, 슬라이더 등

## 5. 간격 시스템

### 기본 간격 단위
- **Base Unit**: `4px`
- **Grid**: 4pt 기반 시스템 추정

### 주요 간격 값
| 용도 | 값 |
|------|-----|
| 컴포넌트 내부 패딩 (소) | `8px` |
| 컴포넌트 내부 패딩 (중) | `12px` ~ `16px` |
| 화면 좌우 여백 | `16px` |
| 카드 간 간격 | `8px` ~ `12px` |
| 섹션 간 간격 | `24px` ~ `32px` |
| 아이콘-텍스트 간격 | `8px` |
| 리스트 아이템 높이 | `56px` ~ `64px` |
| 하단 네비게이션 높이 | `~83px` (세이프 에어리어 포함) |

## 6. 인터랙션 패턴

### 화면 전환
- **Push Navigation**: 상세 페이지로 이동 시 좌→우 슬라이드
- **모달 Present**: 업로드, 설정 등 보조 화면은 하단에서 올라오는 형태
- **탭 전환**: 즉각적 전환, 애니메이션 최소화

### 스크롤
- **무한 스크롤**: 홈 피드에서 콘텐츠 연속 로드
- **상단 고정 헤더**: 네비게이션 바, 검색바 등
- **패럴랙스**: 미디어 카드에서 스크롤 시 이미지 패럴랙스 효과 추정

### 바텀시트 열림/닫힘
- **트리거**: 특정 버튼 탭 또는 상향 스와이프
- **드래그**: 핸들 바를 드래그하여 높이 조절
- **닫기**: 하향 스와이프 또는 외부 영역 탭
- **스냅 포인트**: 절반(Half) / 전체(Full) 두 단계

### 탭 전환
- 하단 탭 탭 → 즉각적 콘텐츠 전환
- 활성 탭 아이콘/텍스트 색상 변경으로 상태 표시

### 미디어 인터랙션
- 이미지/비디오 탭 → 전체화면 뷰어 전환
- 좌우 스와이프로 미디어 탐색
- 핀치-줌 지원 추정

## 7. 디자인 특징

### 전반적인 톤앤매너
- **키워드**: 감성적, 미니멀, 다크 럭셔리, 콘텐츠 중심
- **분위기**: 고급스럽고 절제된 다크 UI, 콘텐츠(사진/영상)가 주인공
- **감성**: 추억, 기억, 순간을 저장한다는 서정적 감성과 현대적 UI의 조화

### 특징적인 디자인 요소

1. **풀스크린 미디어 우선**: 콘텐츠(이미지/비디오)가 UI를 압도하는 구성, 크롬 최소화
2. **그라디언트 오버레이**: 미디어 위에 하단 그라디언트 적용, 텍스트 가독성 확보
3. **레드 포인트 컬러**: 레드(`#FF2020` 계열)를 핵심 CTA와 배지에 집중 사용
4. **퍼플 리워드 테마**: 리워드/포인트 관련 섹션에 퍼플~바이올렛 계열 별도 적용
5. **앰버 코인 아이콘**: 리워드 포인트를 앰버/골드 컬러로 시각화
6. **서비스 이용약관 체크리스트**: 조인/가입 시 바텀시트에 항목별 체크박스 UI
7. **아이콘 스타일**: 선형(라인) 아이콘, 심플하고 두께 균일

### 브랜드 아이덴티티
- **앱 아이콘**: 조각상(포트레이트) + 원색 기하 도형(레드/블루/옐로) 콜라주 스타일
  - 고전 예술(조각상)과 현대 디지털(기하 도형)의 대비
  - "기억 저장소"라는 컨셉을 아이콘으로 시각화
- **브랜드명 표기**: "닷슬래시대시" (한국어) / "DotSlashDash" (영문) 병행
- **슬로건**: "인 뷰 기억 저장소" — 시야에 들어온 순간을 저장한다는 메타포
- **REWARD SYSTEM**: 레드 배경 뱃지로 강조, 커뮤니티 참여 유도
- **플랫폼**: WWIT(윗) 플랫폼 내 서비스로 구성 ("WWIT - 윗" 헤더)

### UI 패턴 요약
- 다크 배경 위에 콘텐츠를 부각하는 "갤러리형" 피드
- 하단 네비게이션 + 바텀시트 조합으로 손 닿기 쉬운 영역 활용
- 가입/약관 동의를 바텀시트 내 체크리스트로 처리
- 리워드 시스템을 별도 컬러(퍼플+앰버)로 구분하여 게임화 요소 강화
- 미디어 중심 콘텐츠에 텍스트 오버레이로 정보 밀도 유지

## 8. travel-planner 적용 시 참고사항

### 채택 권장 요소
- **다크 테마**: 야간 여행, 지도 보기 상황에서 눈 피로 감소
- **미디어 우선 카드**: 여행지 사진이 주인공인 피드 구성
- **바텀시트 패턴**: 여행지 상세 정보, 일정 수정 등에 활용
- **레드 강조색**: 긴급 알림(날씨 경보, 일정 변경) 등에 활용
- **앰버/골드**: 추천 장소, 별점, 포인트 시스템에 활용
- **그라디언트 오버레이**: 지도 위 정보 카드, 여행지 카드에 활용

### 차별화 고려 요소
- 실시간 정보(날씨, 교통) 표시를 위한 상태 인디케이터 추가 필요
- 지도 통합 UI를 위한 컬러 팔레트 확장 고려 (지도 라벨 가독성)
- 시간대/일정 표시를 위한 타임라인 컴포넌트 별도 설계 필요
