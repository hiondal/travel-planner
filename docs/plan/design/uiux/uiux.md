## 1. 디자인 원칙

### 1-1. 핵심 원칙 5개

| # | 원칙 | 설명 | 적용 예시 |
|:-:|------|------|----------|
| 1 | **한눈에 파악 (Glanceable)** | 여행 중 이동하면서 보는 사용자가 3초 이내에 핵심 정보를 인지할 수 있어야 한다. 상태 배지(초록/노랑/빨강)로 복잡한 데이터를 직관적으로 압축한다. | 일정표의 4색 상태 배지, 브리핑 카드의 한 줄 총평 |
| 2 | **즉시 행동 (Actionable)** | 정보를 확인한 뒤 다음 행동까지 탭 1~2회 이내로 완료할 수 있어야 한다. "열고, 보고, 탭한다" 3단계 UX 원칙을 준수한다. | 대안 카드 탭 한 번으로 일정 반영 (P12), 배지 탭 -> 대안 보기 |
| 3 | **맥락 우선 (Context-First)** | 사용자의 현재 맥락(시간, 위치, 상태)에 따라 가장 관련 높은 정보를 먼저 보여준다. 불필요한 정보는 숨기고 필요할 때 펼친다. | 출발 전 브리핑 자동 Push, 위험 항목 우선 표시, 바텀시트로 상세 정보 계층화 |
| 4 | **신뢰 투명성 (Transparent Trust)** | 실시간 데이터의 신선도와 한계를 투명하게 보여준다. 데이터 부재 시 "모르겠다"를 솔직하게 표현한다. | 회색 배지 + "데이터 미확인" (P6), 캐시 데이터 표시 (P20), 만료 브리핑 안내 |
| 5 | **미디어 몰입 (Media-Immersive)** | 여행지 사진/콘텐츠가 주인공이 되는 다크 배경 기반 UI. 크롬(장식 요소)을 최소화하고 콘텐츠에 집중한다. | 풀 블리드 장소 카드, 그라디언트 오버레이, 다크 테마 |

### 1-2. 디자인 언어

- **톤 앤 매너**: 절제된 다크 럭셔리 + 실용적 정보 전달. 감성보다 기능 중심.
- **브랜드 키워드**: 실시간(Real-time), 안심(Assurance), 최적화(Optimize)
- **무드**: 여행의 설렘을 유지하면서도 돌발 상황에 대한 든든한 안전망

### 2-2. 네비게이션 구조

**하단 탭 바 (Bottom Tab Bar) -- 3탭 구성**

| 순서 | 탭명 | 아이콘 | 화면 | 비고 |
|:----:|------|--------|------|------|
| 1 | 일정 | `calendar-days` | 일정표 (배지 포함) | 기본 탭, S05 핵심 |
| 2 | 브리핑 | `bell` | 브리핑 목록 + 대안 카드 | S04/S06 핵심, 미읽음 뱃지 표시 |
| 3 | 마이 | `user` | 마이페이지 | 프로필, 구독, 설정 |

> 탭 수를 3개로 제한하여 핵심 기능에 집중한다. 닷슬래시대시의 5탭과 달리 여행 도중 빠른 접근이 필요한 기능만 탭으로 노출한다.

**상단 네비게이션 바 (Top Navigation Bar)**

| 요소 | 좌측 | 중앙 | 우측 |
|------|------|------|------|
| 메인 화면 | 여행명 (드롭다운) | - | 날짜 선택 |
| 상세 화면 | 뒤로가기 (chevron-left) | 화면 타이틀 | 컨텍스트 액션 |
| 모달/바텀시트 | - | 타이틀 | 닫기 (X) |

- **하단 탭 바**: 일정(활성), 브리핑(미읽음 뱃지 *), 마이
- **좌 스와이프**: 장소 카드 좌 스와이프 시 삭제 버튼 노출 (UFR-SCHD-060)
- **길게 누르기**: 드래그 앤 드롭으로 순서 변경 모드 진입 (UFR-SCHD-070, Could)

**핵심 요소 설명**:
- **바텀시트 패턴**: 핸들 바 드래그로 절반/전체 높이 전환
- **종합 배지**: 상단에 크게 표시, 판정 결과 한눈에 파악
- **4항목 리스트**: 영업상태, 혼잡도, 날씨, 이동시간 각각 개별 상태 아이콘
