## 2. 컬러 팔레트

### 2-1. 배경 컬러 (Dark Theme)

| 역할 | 토큰명 | HEX | 용도 |
|------|--------|-----|------|
| 최상위 배경 | `--bg-primary` | `#0A0A0A` | 앱 전체 배경 |
| 카드/패널 배경 | `--bg-card` | `#1A1A1A` | 카드, 바텀시트, 패널 |
| 입력/구분 배경 | `--bg-input` | `#242424` | 입력 필드, 검색바, 구분 영역 |
| 호버/프레스 배경 | `--bg-hover` | `#2A2A2A` | 탭 가능 영역의 호버/프레스 상태 |
| 오버레이 | `--bg-overlay` | `rgba(0,0,0,0.6)` | 모달/바텀시트 배경 딤 |

### 2-2. 텍스트 컬러

| 역할 | 토큰명 | HEX | 대비비 (vs #0A0A0A) |
|------|--------|-----|:-------------------:|
| 기본 텍스트 | `--text-primary` | `#F0F0F0` | 18.3:1 |
| 보조 텍스트 | `--text-secondary` | `#8A8A8A` | 5.1:1 |
| 비활성 텍스트 | `--text-disabled` | `#555555` | 2.8:1 (장식용) |
| 반전 텍스트 | `--text-inverse` | `#0A0A0A` | - |

### 2-3. 상태 배지 컬러 (S05 핵심)

| 상태 | 토큰명 | HEX | 아이콘 | 의미 | 대비비 (vs #1A1A1A) |
|------|--------|-----|--------|------|:-------------------:|
| 정상 | `--status-green` | `#34C759` | 체크(check) | 모든 항목 정상 | 4.6:1 |
| 주의 | `--status-yellow` | `#FFD60A` | 느낌표(!) | 1개 이상 주의 | 9.5:1 |
| 위험 | `--status-red` | `#FF3B30` | X(close) | 1개 이상 위험 | 4.5:1 |
| 미확인 | `--status-gray` | `#8E8E93` | 물음표(?) | 데이터 수집 불가 | 4.1:1 |

> WCAG 2.1 AA 준수: 모든 상태 배지는 색상 + 아이콘 조합으로 색약 사용자를 지원한다 (NFR-UX-010).

### 2-4. 기능 강조 컬러 (Accent)

| 역할 | 토큰명 | HEX | 용도 |
|------|--------|-----|------|
| CTA/긴급 | `--accent-red` | `#FF2D2D` | 프라이머리 CTA 버튼, 긴급 알림 뱃지 |
| 추천/별점 | `--accent-amber` | `#FFB830` | 평점 별, 추천 강조, 코인/포인트 |
| 프리미엄/구독 | `--accent-purple` | `#7B3FE0` | Pro 뱃지, Paywall, 구독 관련 |
| 정보/링크 | `--accent-blue` | `#0A84FF` | 하이퍼링크, 정보성 안내 |

### 2-5. 그라디언트

| 용도 | 값 | 설명 |
|------|-----|------|
| 카드 하단 오버레이 | `linear-gradient(transparent 0%, rgba(0,0,0,0.7) 100%)` | 이미지 위 텍스트 가독성 확보 |
| 프리미엄 배경 | `linear-gradient(135deg, #7B3FE0 0%, #4A1FB0 100%)` | 구독/Paywall 섹션 |

## 3. 타이포그래피

### 3-1. 폰트 패밀리

```
--font-primary: 'Pretendard', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'SF Mono', 'Fira Code', monospace;
```

> Pretendard: 한/영 혼용 지원, 가변 폰트로 다양한 굵기 커버. iOS에서는 SF Pro로 자동 폴백.

### 3-2. 크기 계층

| 레벨 | 토큰명 | 크기 | 굵기 | 행간 | 용도 |
|------|--------|:----:|:----:|:----:|------|
| Display | `--font-display` | 28px | 700 (Bold) | 36px (1.29) | 온보딩 메인 타이틀, 히어로 텍스트 |
| H1 | `--font-h1` | 22px | 700 (Bold) | 28px (1.27) | 화면 타이틀 |
| H2 | `--font-h2` | 18px | 600 (SemiBold) | 24px (1.33) | 섹션 타이틀, 카드 타이틀 |
| H3 | `--font-h3` | 16px | 600 (SemiBold) | 22px (1.38) | 서브 타이틀, 리스트 타이틀 |
| Body1 | `--font-body1` | 15px | 400 (Regular) | 22px (1.47) | 본문 텍스트, 설명 |
| Body2 | `--font-body2` | 14px | 400 (Regular) | 20px (1.43) | 보조 본문, 카드 설명 |

| Overline | `--font-overline` | 11px | 600 (SemiBold) | 14px (1.27) | 뱃지 레이블, 카테고리 태그 |

### 3-3. 굵기 패턴

| 굵기 | 토큰명 | 값 | 용도 |
|------|--------|:--:|------|

## 5. 아이콘

### 5-1. 아이콘 라이브러리

**Lucide Icons** (오픈소스, MIT 라이선스)

> 선정 이유: 선형(라인) 스타일로 닷슬래시대시의 미니멀 아이콘 톤과 일치, 일관된 스트로크 두께(2px), 모바일 가독성 우수, React Native / Web 모두 지원.

### 5-2. 아이콘 크기

| 크기 | 값 | 용도 |
|------|:---:|------|
| Small | 16px | 인라인 아이콘, 배지 내 아이콘 |
| Medium | 20px | 리스트 아이콘, 버튼 내 아이콘 |
| Default | 24px | 네비게이션 탭, 표준 아이콘 |
| Large | 32px | 상태 아이콘, 빈 상태 일러스트 |
| XLarge | 48px | 온보딩 일러스트, 히어로 아이콘 |

### 5-3. 상태 배지 아이콘 매핑

| 상태 | 아이콘 | Lucide 아이콘명 | 컬러 |
|------|--------|----------------|------|
| 정상 (초록) | 체크 | `check-circle` | `--status-green` |
| 주의 (노랑) | 느낌표 | `alert-triangle` | `--status-yellow` |
| 위험 (빨강) | X | `x-circle` | `--status-red` |
| 미확인 (회색) | 물음표 | `help-circle` | `--status-gray` |

### 5-4. 네비게이션 아이콘 매핑

| 탭 | 아이콘 (비활성) | 아이콘 (활성) | Lucide 아이콘명 |
|----|----------------|--------------|----------------|
| 일정 | 라인 | 화이트 채움 | `calendar-days` |
| 브리핑 | 라인 | 화이트 채움 | `bell` |
| 마이페이지 | 라인 | 화이트 채움 | `user` |

## 8. 인터랙션 디자인

### 8-1. 애니메이션 원칙

| 원칙 | 설명 |
|------|------|
| 목적성 | 모든 애니메이션은 사용자의 행동에 대한 피드백 또는 맥락 전환을 돕는 목적이 있어야 한다 |
| 자연스러움 | 물리 법칙에 기반한 자연스러운 움직임. 가속-감속 커브 적용 |
| 간결함 | 불필요한 장식 애니메이션 금지. 300ms 이내로 완료 |
| 일관성 | 동일 유형의 전환에는 동일한 애니메이션 적용 |

### 8-2. 애니메이션 토큰

| 토큰명 | Duration | Easing | 용도 |
|--------|:--------:|--------|------|
| `--anim-fast` | 150ms | `ease-out` | 버튼 프레스, 토글, 리플 |
| `--anim-normal` | 250ms | `ease-in-out` | 화면 전환, 카드 확장 |
| `--anim-slow` | 350ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 바텀시트 열림/닫힘, 모달 |
| `--anim-spring` | 400ms | `cubic-bezier(0.68, -0.55, 0.27, 1.55)` | 배지 상태 변경, 강조 효과 |

### 8-3. 화면 전환

| 전환 유형 | 애니메이션 | Duration |
|----------|-----------|:--------:|
| Push (상세 진입) | 좌 -> 우 슬라이드 | 250ms |
| Pop (뒤로가기) | 우 -> 좌 슬라이드 | 250ms |
| 바텀시트 열림 | 하단에서 위로 슬라이드 + 배경 딤 | 350ms |
| 바텀시트 닫힘 | 위에서 하단으로 슬라이드 + 딤 해제 | 300ms |
| 모달 열림 | 스케일 0.9 -> 1.0 + 페이드 인 + 배경 딤 | 350ms |
| 탭 전환 | 즉각 전환 (애니메이션 없음) | 0ms |

### 8-4. 마이크로 인터랙션

| 인터랙션 | 동작 | 피드백 |
|----------|------|--------|
| **버튼 탭** | 프레스 상태 | 배경 밝기 10% 증가, 스케일 0.97 -> 1.0 (150ms) |
| **카드 탭** | 대안 카드 선택 | 카드 테두리 `--accent-red` 하이라이트, 체크 아이콘 표시 (250ms) |
| **배지 상태 변경** | 초록->노랑, 노랑->빨강 | 0.8 -> 1.2 -> 1.0 스케일 바운스 + 색상 전환 (400ms) |
| **풀 투 리프레시** | 일정표 상단 당기기 | 로딩 스피너 회전 + 배지 데이터 갱신 |
| **스와이프 삭제** | 리스트 아이템 좌 스와이프 | 빨강 배경 노출 + 삭제 아이콘 |
| **드래그 앤 드롭** | 장소 순서 변경 (C등급) | 그림자 강조 + 투명도 0.7 + 드롭 위치 프리뷰 |
| **로딩 상태** | API 호출 대기 | 스켈레톤 UI (배경색 펄스 애니메이션, 1.5초 주기) |
| **토스트 알림** | 성공/오류 피드백 | 하단에서 슬라이드 업 (250ms) -> 3초 유지 -> 페이드 아웃 (200ms) |
| **온보딩 스와이프** | 좌우 카드 전환 | 수평 슬라이드 + 인디케이터 도트 활성화 |

### 8-5. 제스처 정의

| 제스처 | 대상 | 동작 |
|--------|------|------|
| 탭 | 버튼, 카드, 배지 | 기본 선택/실행 |
| 길게 누르기 (500ms) | 일정 리스트 아이템 | 삭제/편집 메뉴 표시 |
| 좌우 스와이프 | 온보딩 카드, 대안 카드 | 이전/다음 카드 전환 |
| 좌 스와이프 | 일정 리스트 아이템 | 삭제 액션 노출 |
| 상하 스크롤 | 전체 화면 | 콘텐츠 스크롤 |
| 하향 스와이프 | 바텀시트 | 닫기 |
| 풀 투 리프레시 | 일정표 상단 | 데이터 갱신 |
| 드래그 앤 드롭 | 일정 리스트 아이템 | 순서 변경 (UFR-SCHD-070, Could) |

## 9. 접근성 가이드라인 (WCAG 2.1 AA)

### 9-1. 색상 대비

| 대상 | 최소 대비비 | 현재 준수 |
|------|:---------:|:---------:|
| 일반 텍스트 (Body) | 4.5:1 | `--text-primary` (#F0F0F0 on #0A0A0A) = 18.3:1 |
| 큰 텍스트 (H1, H2) | 3:1 | `--text-primary` = 18.3:1 |
| 보조 텍스트 | 4.5:1 | `--text-secondary` (#8A8A8A on #0A0A0A) = 5.1:1 |
| UI 컴포넌트 | 3:1 | 모든 상태 배지 색상 >= 4.1:1 |
| 상태 배지 (카드 위) | 3:1 | 모든 배지 색상 on #1A1A1A >= 4.1:1 |

### 9-2. 접근성 체크리스트

- [x] 색상만으로 정보를 전달하지 않는다 (상태 배지: 색상 + 아이콘 조합)
- [x] 터치 타겟 최소 44x44px

## 10. 다크 테마 적용 원칙

| 원칙 | 설명 |
|------|------|
| 순수 블랙 배경 | `#0A0A0A`를 최상위 배경으로 사용. 순수 `#000000`은 OLED 번인 방지를 위해 회피 |
| 레이어 기반 깊이 | 그림자 대신 배경색 밝기 차이로 깊이를 표현 (섹션 7 참조) |
| 이미지 가독성 | 이미지 위 텍스트는 반드시 그라디언트 오버레이 적용 |
| 하이 콘트라스트 텍스트 | 기본 텍스트는 `#F0F0F0` (순수 화이트 `#FFFFFF` 대신 약간 낮춰 눈부심 방지) |
| 강조색 채도 조절 | 다크 배경 위에서 시각적 균형을 위해 강조색 채도를 90% 수준으로 조절 |
| 미디어 우선 | 콘텐츠(여행지 사진)가 UI를 압도하는 구성. 크롬(장식 요소) 최소화 |
