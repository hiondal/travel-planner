## 2. 정성 데이터 분석

### 2-1. 문제가설별 핵심 인사이트

#### 문제가설 A: 대안 탐색 시간 소모

**공통 경험 패턴 (3명 반복)**

1. **"앱 삼각 동선"의 보편성**: 3명 모두 구글맵, 블로그(네이버), 인스타그램, 트리플 등 3개 이상의 앱을 번갈아 열며 대안을 탐색했다. 각 앱이 제공하는 정보(위치/거리, 한국어 리뷰, 비주얼, 실시간 상태)가 분절되어 있어 "하나로 다 되는 게 없다"는 인식이 공통적이다.

2. **검색 시간 30분 내외**: 3명 모두 돌발 상황 발생 후 대안을 확정하기까지 30분 내외를 소요했다. S1은 "30분 넘게", S2는 "약 30분", S3은 "30분 넘게"라고 응답하여 23.5분이라는 기존 가설보다 체감 시간이 더 길다.

3. **검색 피로에 의한 경험 품질 하락**: 3명 모두 검색에 시간과 에너지를 소모한 뒤 도착한 대안 장소에서도 제대로 즐기지 못했다고 증언. 검색 노동은 시간 손실에 그치지 않고 이후 경험의 질까지 저하시킨다.

**세그먼트별 차별화 포인트**

| 차별화 요소 | S1 솔로 | S2 그룹 | S3 초보 |
|-----------|---------|---------|---------|
| 검색 행동 | 효율적/체계적으로 검색하지만 "최선인지" 확신 부족으로 반복 비교 | 4명이 각자 검색 → 서로 다른 결과로 논쟁 발생 | 정보 판단 기준 자체가 없어 블로그 연도, 리뷰 신뢰도 분별 못 함 |
| 핵심 불편 | 시간당 가치 환산 → "30분 = 돈" | 검색 분산이 아닌 증폭 → "합의 비용" 추가 발생 | 검색 능력 자체의 부족 → "가이드 부재" |
| 원하는 해결 | 맥락 기반 대안 2~3개 + 추천 이유 | 대안 공유 + 그룹 빠른 합의 구조 | 패키지 가이드처럼 "이렇게 하세요" 안내 |

**새롭게 발견된 인사이트**

- **그룹 검색의 역설**: 4명이 검색하면 빨라질 것 같지만, 각자 다른 정보를 가져와 "여기 가자/아니 여기가 낫다"로 논쟁이 시작되어 오히려 더 느려진다 (S2). 인원수가 많을수록 합의 비용이 기하급수적으로 증가하는 구조.
- **취향 분리 니즈**: 동행자 간 취향이 다를 때 "잠깐 나뉘었다 합류"하는 옵션 추천 니즈 발견 (S2). 그룹 여행에서 개인화의 가능성.
- **정보 판별력 격차**: S1(경험 많음)은 블로그 연도, 리뷰 신뢰도를 판별하지만 S3(초보)은 2023년 블로그도 "현재 유효"로 간주. 정보 리터러시 수준에 따른 UX 차별화 필요.

#### 문제가설 B: 사후 인지

**공통 경험 패턴 (3명 반복)**

1. **"도착해서야 알게 되는" 무력감의 보편성**: 3명 모두 15~30분의 이동 시간을 투자한 뒤 목적지에서 휴업/조기 마감/혼잡을 발견한 경험이 있다. 이때의 감정을 S1은 "무력감", S2는 "멘붕", S3은 "멘붕"으로 표현. 분노가 아닌 무력감이라는 점이 공통적이다.

2. **플랫폼 정보 불신의 축적**: 3명 모두 구글맵 "영업 중" 표시가 실제와 다른 경험을 했다. 이 경험이 반복되면 도구 자체에 대한 불신이 축적되며, S3처럼 "블로그에 매일 영업이라고 써있었다"는 증언은 정보 소스의 신뢰성 문제를 구조적으로 보여준다.

3. **사후 인지의 감정 전이 효과**: 3명 모두 사후 인지 한 번이 남은 반나절 이상의 감정에 부정적 영향을 미쳤다고 증언. S1은 "여행 전체 기분을 망침", S2는 "오후 일정 전체가 우울", S3은 "오후에 사진도 별로 안 찍었다". 단일 사건의 감정 반감기가 매우 길다.

**세그먼트별 차별화 포인트**

| 차별화 요소 | S1 솔로 | S2 그룹 | S3 초보 |
|-----------|---------|---------|---------|
| 감정 처리 | 혼자서 부정적 감정을 삭여야 함 → "감정 고립" | 특정인에게 책임 전가 → "관계 훼손" | 자유여행 자체에 대한 자기 의심 유발 |
| 영향 범위 | 본인 1명의 여행 만족도 | 그룹 4명 전체의 분위기 + 관계 | 향후 여행 방식 결정(자유여행 재시도 여부) |
| 핵심 가치 | 시간 매몰 비용 방지 | 그룹 신뢰 유지 + "누구의 잘못도 아님" | 패키지→자유여행 전환 장벽 제거 |

**새롭게 발견된 인사이트**

- **솔로 여행자의 "감정 고립"**: 동행이 있으면 "다른 데 가자~"로 분위기가 전환되지만, 솔로는 부정적 감정을 혼자 삭이면서 다음 의사결정까지 혼자 해야 한다 (S1). 기존 가설에 없던 감정적 차원의 발견.
- **그룹 여행의 "책임 추궁" 역학**: 한 명이 추천한 곳이 문 닫혀 있으면 그 사람이 책임감을 느끼고, 나머지는 은근히 불만을 가진다. 시스템이 알림을 보내면 "누구의 잘못도 아닌" 상황이 되어 관계 보호 기능을 수행한다 (S2).
- **첫 자유여행의 "성공 경험" 중요성**: 사후 인지의 반복은 S3에게 "자유여행이 나에게 안 맞나"라는 자기 의심을 유발. 반대로 사전 알림이 있으면 "AI가 가이드 역할을 대체"한다는 인식이 되어, 패키지→자유여행 전환 장벽을 낮추는 가장 강력한 가치 제안이 된다 (S3).
- **SNS 콘텐츠 손실**: 기대했던 장소를 못 가면 인스타그램 등에 올릴 "인생샷"을 못 찍어서 이중으로 아쉽다는 반응 (S2). 여행 만족도의 구성 요소로 SNS 기록 가치가 포함됨.

#### 문제가설 C: 연쇄 일정 붕괴

**공통 경험 패턴 (3명 반복)**

1. **도미노 효과의 보편적 경험**: 3명 모두 일정 1개 변경 시 후속 2~3개 일정에 연쇄 영향이 발생한 경험이 있다. S1은 "후시미이나리→기요미즈데라→니시키시장" 순서 붕괴, S2는 "후시미이나리→니시키시장→저녁식당" 연쇄 변경, S3은 "로프웨이→오와쿠다니→유람선→신사" 전체 붕괴.

2. **수동 재계산의 스트레스**: 3명 모두 길거리에서 서서/더운 날/비 오는 날에 수동으로 이동시간, 영업시간, 순서를 재계산한 경험이 있다. 물리적 환경 스트레스와 인지적 부하가 동시에 발생하는 "감정-인지 미스매치" 상황.

3. **"자동으로 해주는 게 있었으면"이라는 자발적 니즈 표현**: S1은 "이걸 자동으로 해주는 게 있었으면", S3은 "자동으로 다시 짜주는 게 있으면 얼마나 좋을까"라고 각각 자발적으로 언급. 솔루션 니즈가 인터뷰어의 유도 없이 자연 발생.

**세그먼트별 차별화 포인트**

| 차별화 요소 | S1 솔로 | S2 그룹 | S3 초보 |
|-----------|---------|---------|---------|
| 일정 스타일 | 시간 단위 빡빡한 일정 → 붕괴 시 도미노 심각 | 느슨한 큰 틀 → 완전 붕괴는 아님 | 시간 단위 일정이나 현지 변수 미반영 |
| 핵심 병목 | 대안 선택의 "확신 부족" | 대안에 대한 "그룹 합의" 시간 | 현지 재계산 "능력 부족" |
| 원하는 UX | 대안 2~3개 + 변경 이유 설명 | 대안 + 장단점 비교 + 그룹 투표 | 전자동 재조정 (가이드 수준) |
| 부분동의 이유 | "감지(B)가 선행되어야 재최적화(C)가 의미" | "자동 재조정보다 그룹 합의가 우선" | (동의 -- 해당 없음) |

**새롭게 발견된 인사이트**

- **B→C 의존 관계 명확화**: S1이 명시적으로 "감지 없이 재최적화만 있으면 소용없다"고 지적. 문제 B(사후 인지)의 해결이 문제 C(연쇄 붕괴)의 선행 조건이라는 인과관계가 사용자 관점에서도 확인됨.
- **세그먼트별 재조정 UX의 근본적 차이**: 솔로는 "대안 선택형", 그룹은 "합의(투표)형", 초보는 "가이드(자동)형"으로 같은 기능이라도 전혀 다른 인터랙션 패턴이 필요. 단일 UX로 해결 불가.
- **완전 자동에 대한 불안**: S1은 "왜 이렇게 바꿨지? 이유를 모르면 신뢰가 안 된다"고 지적. 자동 재조정의 투명성(변경 이유 설명)이 신뢰의 핵심 조건.
- **일정 빡빡함과 기능 가치의 정비례**: 빡빡한 일정을 짜는 사람일수록 연쇄 붕괴가 심각하고, 따라서 이 기능의 지불 의향도 높다 (S1 자가 진단).

| #2 윤서연 (S2) | C | "자동 재조정"보다 "변경된 일정안을 그룹원에게 바로 공유하고 투표할 수 있는 기능"이 더 필요. 엔진이 혼자 결정하면 "나는 동의 안 했는데?"라는 반발 발생 | 그룹 여행 UX에서는 "자동 재조정 + 그룹 합의" 결합이 필수 |

#### 추가로 제시된 근본원인 및 조건

1. **감지-재조정 파이프라인 조건** (S1): 상황 변화 감지(B) 없이 재최적화(C)만 있으면 소용없다. B의 해결이 C의 전제 조건이다.

2. **그룹 합의 메커니즘** (S2): AI가 대안 3개를 제시하고, 각 옵션의 장단점(이동 시간, 비용, 리뷰 점수)을 보여준 뒤 그룹원이 실시간으로 투표하는 구조가 필요하다.

3. **자동화 수준의 조절** (S1): 완전 자동은 불안을 유발. 대안 2~3개 제시 + 변경 이유 설명 + 사용자 선택이라는 "반자동" 방식이 신뢰를 형성한다.

4. **일정 의존 관계의 다차원성** (S3/인사이트): 교통 연결, 영업시간 제약, 지리적 클러스터링, 예약 유무, 체력 소모도 등을 모두 고려해야 하는 복합 최적화 문제. 단순 시간순 재배치로는 해결 불가.

## 4. 핵심 인사이트 Top 5

### 1위: 문제 B(사후 인지)는 오류 연쇄의 트리거이자 만장일치 최심각 문제

3명 전원이 가장 심각한 문제로 B를 선택했으며, 중요도 5.0 + 불편도 5.0 + 동의율 100%로 가장 강력하게 검증되었다. B가 발생하면 A(대안 탐색)가 강제되고, A의 결과가 C(연쇄 붕괴)를 유발하는 인과 체인이 사용자 관점에서도 명확히 인식되고 있다. **B의 해결이 전체 문제 구조의 레버리지 포인트다.**

### 2위: 세그먼트별로 "같은 문제, 다른 고통"이 존재한다

동일한 문제(B: 사후 인지)라도 고통의 양상이 근본적으로 다르다:
- **솔로(S1)**: 감정 고립 -- 부정적 감정을 전환해줄 동행이 없어 오롯이 혼자 감당
- **그룹(S2)**: 책임 추궁 -- "왜 미리 확인 안 했어?"라는 비난이 관계를 훼손
- **초보(S3)**: 자기 의심 -- "자유여행이 나에게 안 맞나"라는 여행 방식 자체에 대한 회의

이 차별화는 동일한 Push 알림이라도 세그먼트별로 다른 메시지 톤과 후속 UX가 필요함을 시사한다.

### 3위: 문제 C의 해결 방식은 세그먼트별로 근본적으로 다르다

문제 C(연쇄 붕괴)의 동의율이 66.7%에 그친 이유는 "자동 재최적화 엔진"이라는 단일 근본원인이 세그먼트별 니즈를 포괄하지 못하기 때문이다:
- **솔로(S1)**: 대안 2~3개 + 변경 이유 설명 + 내가 선택 (반자동)
- **그룹(S2)**: 대안 + 장단점 비교 + 그룹 투표 (합의형)
- **초보(S3)**: "이렇게 하세요" 수준의 가이드 (전자동)

**단일 UX로 3개 세그먼트를 만족시킬 수 없다.** 세그먼트별 재조정 모드(선택형/투표형/가이드형)가 설계에 반영되어야 한다.

### 4위: 그룹 여행에서 "합의 비용"은 기존 가설에 없는 새로운 병목이다

S2(그룹)의 인터뷰에서 반복적으로 드러난 패턴: 검색 결과 논쟁(20분), 일정 변경 합의(20분), 식당 재예약(15분) 등 "그룹 합의"에 소요되는 시간이 개인 검색 시간과 별도로 발생한다. 카카오톡 기반의 비구조적 논의가 병목의 원인이며, "대안 제시 + 실시간 투표"라는 구조적 합의 도구가 그룹 여행의 핵심 차별화 기능이 될 수 있다.

### 5위: 정보 정확도 90% 이상이 서비스 신뢰의 최소 기준선이다

S1이 명시적으로 "정확도 90% 이상은 보장되어야 신뢰가 쌓인다"고 언급했으며, 3명 모두 구글맵/블로그의 정보 부정확성이 사후 인지의 직접적 원인이라고 지적했다. 우리 서비스가 동일한 문제를 반복하면 기존 도구 대비 차별성이 없다. **정확도는 기능이 아니라 존재 이유(reason to exist)다.**

(3) 재조정 결과의 투명성/설명 가능성 부재
    → "왜 이렇게 바꿨지?" 이유를 모르면 신뢰 불가 (S1)
