# 5WHY 분석 -- 도그냥 (서비스 기획자)

> 분석 관점: 고객 경험 흐름, 서비스 이용 동선, 사용자 행동 패턴
> 데이터 기반: 인터뷰 5명, 관찰 10회, 체험 10회, 종합 분석 결과

## 현상문제 1: 돌발 상황 발생 시 대안을 찾는 데 평균 23.5분이 소요되어, 여행 시간이 직접적으로 손실된다

> 여행자가 예상치 못한 상황(휴업, 날씨 변화, 혼잡 등)을 마주한 후 대안 장소를 찾기까지 평균 23.5분(최소 12분~최대 40분)이 걸리며, 3박 4일 여행 기준 매일 반복되면 하루 일정 하나 분량(약 2시간)이 "검색 노동"에 소모된다. (인터뷰 5/5명, 체험 10/10회 확인. 심각도 5.0/5)

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| WHY1 | 왜 대안을 찾는 데 23.5분이나 걸리는가? --> 여행자가 거리, 리뷰, 영업 상태, 사진이라는 4가지 판단 기준을 충족하는 장소를 찾기 위해 구글맵, 네이버 블로그, 인스타그램을 번갈아 오가는 "삼각 동선" 탐색을 수행하기 때문이다. (인터뷰 4/5명이 3개 이상 앱 전환 패턴 확인. 소윤: "인스타에서 찾고, 블로그에서 확인하고, 구글맵에서 위치 잡고... 이 삼각 동선이 진짜 피곤해요.") |
| WHY2 | 왜 여러 앱을 번갈아 오가야 하는가? --> 현재 시장에 거리+한국어 리뷰+실시간 영업 상태+비주얼을 한 화면에서 비교할 수 있는 통합 서비스가 존재하지 않기 때문이다. 구글맵은 거리 정보는 강하지만 한국어 리뷰가 부족하고, 네이버는 한국어 리뷰는 풍부하지만 위치/영업 실시간 정보가 약하며, 인스타그램은 비주얼은 좋지만 위치 정확도와 영업시간 정보가 없다. |
| WHY3 | 왜 "여행 중 실행 단계"를 위한 통합 서비스가 없는가? --> 기존 여행 서비스(트리플, 마이리얼트립 등)가 "여행 전 계획" 단계에 집중하고 있으며, "여행 중 돌발 대응"이라는 사용 맥락(현재 위치 + 현재 상황 + 즉시 판단)을 핵심 시나리오로 설계한 서비스가 없기 때문이다. (준혁: "트리플은 여행 전에 썼는데, 현지에서는 구글맵이 더 편해서 안 쓰게 돼요.") |
| WHY4 | 왜 기존 서비스들은 "여행 중 실행"을 핵심 시나리오로 다루지 않았는가? --> 여행 중 실시간 대응은 현재 위치, 실시간 영업 상태, 날씨, 혼잡도, 사용자 취향을 동시에 결합해야 하는 다차원 맥락 처리가 필요한데, 이를 기술적으로 구현하기 어려웠고, "여행 전 계획" 시장이 더 크고 수익화가 명확했기 때문이다. |
| WHY5 | 왜 다차원 맥락 처리가 서비스화되지 못했는가? --> "여행 중 돌발 상황"이라는 순간은 짧고 급박하여 사용자 행동 데이터를 수집하기 어렵고, 실시간 영업/혼잡 데이터를 확보하는 비용이 높아 기존 플랫폼이 투자 대비 수익을 정당화하지 못했기 때문이다. 그러나 AI 기반 맥락 추론과 다중 데이터 소스 통합이 가능해진 현재, 이 문제를 해결할 기술적 조건이 갖춰지고 있다. |

### 다층적 근본원인 검토

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | "최선의 선택을 해야 한다"는 완벽주의적 탐색 행동. 혼자 여행 시 의사결정을 검증해줄 상대가 없어 후보를 반복 비교하며 확신을 얻으려 함 (인터뷰 5/5명 "이게 최선인지 모르겠다") | ★★★ |
| 제도/시스템적 | 여행 중 실행 단계를 위한 통합 정보 허브의 부재. 거리+리뷰+영업상태+사진을 한 화면에 제공하는 서비스가 없어 앱 간 삼각 동선이 구조적으로 발생 | ★★★★★ |
| 맥락적 | 돌발 상황은 "지금, 여기서, 즉시"라는 시간-공간-긴급도의 삼중 제약 하에서 발생하나, 기존 검색 도구는 정적 정보(과거 리뷰, 고정 영업시간)만 제공하여 맥락 불일치가 발생 | ★★★★ |
| 커뮤니케이션 | 현지어 리뷰(일본어, 태국어, 프랑스어)를 한국어로 접근할 수 없어 판단 근거의 절반 이상이 차단됨. 동행자 간 서로 다른 앱에서 다른 기준으로 검색하여 정보 공유가 단절됨 | ★★★ |

**핵심 근본원인**: **"여행 중 실행 단계"를 위한 통합 정보 허브의 구조적 부재** -- 거리, 한국어 리뷰, 실시간 영업 상태, 비주얼을 하나의 화면에서 비교 판단할 수 있는 서비스가 없어, 여행자가 매 돌발 상황마다 3개 이상의 앱을 오가는 비효율적 탐색을 반복해야 한다. 이는 개인의 탐색 능력 문제가 아니라, 서비스 생태계의 구조적 공백이다.

## 현상문제 2: 여행자의 80%가 돌발 상황을 "도착한 후에야" 인지하여, 이동 시간이 매몰 비용이 되고 감정 곡선이 급락한다

> 인터뷰 4/5명이 휴업, 혼잡, 교통 중단 등의 돌발 상황을 목적지에 도착한 후에야 인지하는 "사후 인지" 패턴을 보인다. 이로 인해 이동에 투자한 15~75분이 매몰 비용이 되고, 감정 곡선이 3.6에서 1.6으로 2.0점 급락하는 "감정 절벽"이 발생한다. (심각도 4.8/5. 관찰 6/10회, 체험 8/10회 확인)

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| WHY1 | 왜 여행자가 도착해서야 돌발 상황을 인지하는가? --> 이동 중에 도착 예정 장소의 실시간 상태(임시 휴업, 혼잡도, 교통 중단, 날씨 변화)를 사전에 확인할 수 있는 알림 시스템이 없기 때문이다. 여행자는 출발 전 정적 정보(구글맵 영업시간)만 확인하고 이동을 시작한다. |
| WHY2 | 왜 이동 중 실시간 상태 알림이 없는가? --> 현재 여행자가 사용하는 도구(구글맵, 트리플 등)는 "일정에 등록된 장소의 상태를 실시간으로 모니터링하여 선제적으로 알림을 보내는" 기능을 제공하지 않기 때문이다. 구글맵의 영업 정보는 사업자가 등록한 정적 데이터이며, 임시 휴업이나 국가 행사에 의한 폐쇄 등 실시간 변수를 반영하지 못한다. (준혁: "구글맵이 '영업 중'이라고 하는 걸 100% 믿을 수 없다는 게 제일 스트레스예요.") |
| WHY3 | 왜 기존 플랫폼의 영업 정보가 부정확한가? --> 영업 정보의 업데이트 책임이 사업자(가게 주인)에게 있으나, 소규모 가게나 비영어권 국가에서는 정보 업데이트가 활발하지 않다. 임시 휴업, 정기 휴일 변경, 국가 행사 등 예외적 상황은 구조적으로 반영되기 어렵다. (관찰5: 구글맵 "토요일 영업" 표시되어 있으나 실제 정기휴일. 체험3: 왕궁 임시 폐쇄 정보가 태국어 공식 홈페이지에만 안내) |
| WHY4 | 왜 여행자는 출발 전에 직접 확인하지 않는가? --> 여행자의 일정 준비 행동은 "구글맵 영업시간 확인 + 블로그 정보 참조"로 완결되는 심리적 체크리스트가 형성되어 있으며, 이 수준의 확인으로 충분하다고 인지하기 때문이다. 특히 현지어(일본어, 태국어)로 된 공식 정보에 접근하는 것은 언어 장벽으로 불가능하고, "전화로 확인"도 언어와 요금 문제로 비현실적이다. |
| WHY5 | 왜 "출발 전 확인이면 충분하다"는 인지가 형성되었는가? --> 국내 여행에서는 네이버 지도/카카오맵의 실시간 정보가 상대적으로 정확하여 "앱에서 영업 중이면 실제도 영업 중"이라는 신뢰 모델이 형성되어 있고, 이 경험이 해외 여행에서도 그대로 전이되기 때문이다. 그러나 해외에서는 데이터 정확도가 현저히 낮아 이 신뢰 모델이 반복적으로 배반당하면서, 디지털 도구 전반에 대한 불신으로 이어진다. |

### 다층적 근본원인 검토

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | 국내 경험에서 형성된 "앱 정보 = 현실"이라는 신뢰 모델이 해외 여행에서 반복적으로 깨지면서, 당황의 강도가 증폭됨. 특히 "계획대로 진행 중"이라는 높은 만족(3.6)에서 갑자기 추락하는 감정 낙차가 충격을 극대화 | ★★★★ |
| 제도/시스템적 | 여행 일정에 등록된 장소의 상태를 실시간 모니터링하고, 이상 감지 시 이동 전에 선제적으로 알리는 "사전 경고 시스템"의 부재. 기존 플랫폼은 사용자가 직접 확인해야 하는 "풀(Pull)" 모델만 제공하고, 시스템이 먼저 알려주는 "푸시(Push)" 모델이 없음 | ★★★★★ |
| 맥락적 | 해외 현지의 임시 휴업, 국가 행사, 교통 점검 등은 현지어 공식 채널에서만 안내되어, 외국인 여행자가 사전에 접근하기 구조적으로 불가능. 정보의 "언어 장벽"이 사후 인지의 직접적 원인 | ★★★★ |
| 커뮤니케이션 | 현지 안내 표지판이 현지어로만 제공되어 도착 후에도 정확한 상황 파악이 지연됨 (관찰3: 태국어/영어 안내문만. 체험6: 스페인어 파업 안내문). 공식 홈페이지의 다국어 지원이 미비하여 사전 확인 자체가 차단됨 | ★★★ |

**핵심 근본원인**: **"여행 일정 기반 선제적 리스크 알림(Push) 시스템"의 부재** -- 현재 모든 여행 도구는 사용자가 직접 정보를 찾아야 하는 Pull 모델이다. 여행자의 일정에 등록된 장소를 실시간으로 모니터링하고, 이동 전에 "이 장소는 현재 임시 휴업입니다", "30분 후 폭우 예상, 실내 일정으로 전환하시겠습니까?"와 같은 선제적 알림을 보내는 Push 모델이 없기 때문에, 여행자는 매번 도착해서야 당황하게 된다.

## 현상문제 3: 하나의 일정 변경이 전체 여행 일정을 연쇄적으로 붕괴시키며, 여행자가 이를 수동으로 재조정하는 데 반복적 인지 부하가 발생한다

> 돌발 상황으로 하나의 장소를 변경하면 이동 동선, 영업시간, 예약 시간, 일몰/일출 시간 등 다수의 의존 관계가 연쇄적으로 틀어지는 "도미노 효과"가 발생한다. 여행자는 이를 머릿속이나 메모장에서 수동으로 재계산해야 하며, 이 과정에서 5~15분의 추가 시간과 높은 인지적 피로가 소모된다. 심한 경우 당일 일정뿐 아니라 여행 전체 일정이 재편된다. (인터뷰 4/5명, 체험 8/10회 확인. 심각도 4.4/5. 소윤: "마지막 날에 원래 첫째 날 일정을 하게 됐어요.")

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| WHY1 | 왜 하나의 변경이 전체 일정을 연쇄적으로 붕괴시키는가? --> 여행 일정의 각 항목이 시간(이전 일정 종료 시각), 위치(이동 동선 순서), 외부 조건(영업시간, 예약, 일몰)에 의해 서로 의존 관계에 있기 때문이다. 하나의 항목이 바뀌면 이후 모든 항목의 시작 시각, 이동 경로, 실행 가능 여부가 동시에 영향을 받는다. (시우: 라멘집 변경 -> 1시간 지연 -> 오호리 공원 일몰 실패. 소윤: 비로 기요미즈데라 연기 -> 아라시야마 포기 -> 마지막 날까지 연쇄 변경) |
| WHY2 | 왜 이 연쇄 변경을 수동으로 처리해야 하는가? --> 여행자가 사용하는 도구(노션, 메모 앱, 트리플, 구글 스프레드시트)는 일정을 "정적 목록"으로만 관리하며, 항목 간 시간-위치 의존 관계를 인식하지 못하기 때문이다. 하나를 바꿔도 나머지는 그대로 남아 있어, 여행자가 직접 모든 항목을 재계산해야 한다. |
| WHY3 | 왜 기존 일정 도구는 의존 관계를 인식하지 못하는가? --> 기존 여행 일정 도구는 "일정표 작성" 기능에 초점을 맞추고 있으며, "일정 실행 중 동적 변경"이라는 사용 시나리오를 핵심으로 설계하지 않았기 때문이다. 일정 간 의존 관계를 계산하려면 각 장소의 영업시간, 위치 좌표, 이동 소요 시간, 예약 조건 등 다층적 데이터를 실시간으로 결합해야 하는데, 이는 단순 목록형 도구의 설계 범위를 초과한다. |
| WHY4 | 왜 "동적 일정 재조정"이 서비스로 구현되지 않았는가? --> 동적 재조정은 "최적화 문제"(여러 제약 조건 하에서 최적의 순서와 시간 배분을 계산)인데, 여행 일정의 제약 조건은 매우 다양하고 개인마다 다르며(아이 동반 시 낮잠 시간, 리피터의 혼잡도 회피 선호 등), 이를 범용적으로 처리하는 것이 기술적으로 어려웠기 때문이다. |
| WHY5 | 왜 여행자 개인이 이 재조정을 머릿속에서 처리하는 것이 문제인가? --> 돌발 상황 직후는 감정적으로 가장 취약한 시점(감정 레벨 1.6~1.9)이며, 이 상태에서 복수의 변수(시간, 거리, 영업, 동선)를 동시에 고려하는 인지적 작업을 수행하는 것은 판단의 질을 저하시키고, 피로를 가중시키며, "이게 최선인가?"라는 확신 부족을 만성화시킨다. 결과적으로 돌발 상황이 해소된 후에도 피로가 잔존하여 나머지 여행의 만족도를 낮춘다. (인터뷰 4/5명 오후 피로 누적 언급. 체험3: "아침 2시간의 멘탈 소모가 하루 전체 피로에 영향") |

### 다층적 근본원인 검토

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | 감정적으로 가장 취약한 시점(1.6~1.9)에서 인지적으로 가장 복잡한 작업(다변수 최적화)을 수행해야 하는 "감정-인지 미스매치". 이로 인해 판단의 질이 저하되고, "이게 최선인지 모르겠다"는 확신 부족이 만성화됨 (5/5명 전원 경험) | ★★★★ |
| 제도/시스템적 | 일정 항목 간 시간-위치-조건 의존 관계를 인식하고, 하나의 변경 시 나머지를 자동으로 재최적화하는 "동적 일정 엔진"의 부재. 현재의 일정 도구는 정적 목록 관리만 가능 | ★★★★★ |
| 맥락적 | 일정 재조정은 당일 일정에만 국한되지 않고 여행 전체(2~7일)에 연쇄 영향을 미침. "마지막 날에 첫째 날 일정을 하게 됨"(소윤)처럼 복수 일에 걸친 도미노 효과는 단일 일 최적화로는 해결 불가. 여행 전체를 조망하는 시야가 필요하나, 돌발 상황의 급박함 속에서 이런 거시적 판단은 현실적으로 불가능 | ★★★★ |
| 커뮤니케이션 | 일정 변경의 연쇄 영향이 여행자에게 투명하게 전달되지 않음. "이 장소를 바꾸면 다음 3개 일정에 이런 영향이 있습니다"라는 시각적 피드백이 없어, 여행자가 변경의 전체 파급 효과를 인지하지 못한 채 의사결정을 함 | ★★★ |

**핵심 근본원인**: **일정 항목 간 의존 관계를 인식하는 "동적 일정 재조정 엔진"의 부재** -- 여행 일정은 시간, 위치, 영업시간, 예약, 날씨 등 다수의 의존 관계로 연결된 복합 시스템이나, 현재의 모든 일정 도구는 이를 독립적인 항목의 나열(정적 목록)로만 관리한다. 하나의 변경이 전체에 미치는 연쇄 영향을 계산하고, 최적의 재배치안을 즉시 제시하는 동적 엔진이 없기 때문에, 여행자는 감정적으로 가장 취약한 순간에 인지적으로 가장 부담이 큰 작업을 수동으로 수행해야 한다.

## 종합: 3개 현상문제의 구조적 관계

```
[현상문제 2: 사후 인지]          [현상문제 1: 탐색 시간 소모]        [현상문제 3: 연쇄 붕괴]
도착해서야 문제를 알게 됨   -->   대안을 찾느라 23.5분 허비   -->   하나를 바꾸면 전체가 무너짐
    |                              |                              |
    v                              v                              v
  감정 절벽 (3.6->1.6)         감정 바닥 유지 (1.9)            회복 지연 (2.5)
    |                              |                              |
    v                              v                              v
 [근본원인]                    [근본원인]                      [근본원인]
 선제적 Push 알림              통합 정보 허브                  동적 일정 재조정
 시스템의 부재                 의 부재                        엔진의 부재
```

세 가지 현상문제는 독립적이 아니라 **시간 순서상 연쇄적으로 발생**한다. 사후 인지(문제2)가 탐색 시간 소모(문제1)를 유발하고, 탐색 결과로 일정을 변경하면 연쇄 붕괴(문제3)가 따라온다. 따라서 서비스 설계 시 세 근본원인을 개별이 아닌 **하나의 연결된 경험 흐름**으로 해결해야 한다.

| 순서 | 근본원인 | 서비스 개입 시점 | 기대 효과 |
|------|---------|----------------|----------|
| 1 | 선제적 Push 알림 시스템의 부재 | 2단계->3단계 전환점 (감정 절벽 이전) | 사후 인지를 사전 인지로 전환, 감정 낙차 -2.0을 -0.8 이내로 완화 |
| 2 | 통합 정보 허브의 부재 | 4단계 (대안 탐색) | 탐색 시간 23.5분->3분 이내, 삼각 동선 제거 |
| 3 | 동적 일정 재조정 엔진의 부재 | 5단계 (대안 선택 및 일정 재조정) | 수동 재계산 제거, 도미노 효과 자동 해소 |
