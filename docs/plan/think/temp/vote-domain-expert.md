## B 투표 (비즈니스 가치) -- 3표

| 솔루션 | 투표수 | 투표 근거 |
|--------|:------:|----------|
| S04 출발 전 브리핑 | 2표 | 여행 산업에서 "사전 브리핑"은 패키지 투어 가이드가 수십 년간 제공해온 핵심 가치다. 투어 가이드가 버스 안에서 "다음 목적지 날씨가 좋지 않아 순서를 바꾸겠습니다"라고 미리 안내하면 여행자의 불만이 급감하는 것은 업계에서 이미 검증된 사실이다. 자유여행에서 이 가이드 역할이 완전히 비어 있다는 점이 During-trip 구조적 공백의 본질이다. S04는 이 공백을 가장 직접적으로 메운다. 또한 6명이 독립 발상했다는 점은 이 가치가 직관적으로 명확하다는 증거다. 건당 과금의 첫 번째 유료 기능으로 전환율이 가장 높을 것으로 판단한다. 여행자가 "출발 전에 알려줬으면 좋겠다"는 것은 인터뷰 3명 전원이 독립적으로 확인한 핵심 니즈이며, WTP 5,000~10,000원의 직접적 과금 근거가 된다. |
| S06 맥락 맞춤 대안 카드 | 1표 | 킹핀 B(사후 인지)를 해결해도 돌발 상황 자체를 100% 차단할 수는 없다. 사전에 알고도 대안이 필요한 상황은 반드시 발생한다. 이때 23.5분의 검색 노동을 3분으로 줄이는 것은 "시간 = 돈"으로 환산하는 효율 추구형 솔로 트래블러에게 즉각적 체감 가치를 제공한다. 여행 산업에서 "큐레이션"의 가치는 이미 마이리얼트립(MAU 500만)이 증명했다. 다만 마이리얼트립의 큐레이션은 Before-trip이고, S06은 During-trip 실시간 맥락 큐레이션이라는 점에서 완전히 새로운 카테고리를 정의한다. A(대안 탐색) 문제를 직접 해결하면서 B와의 연쇄 가치를 극대화하는 솔루션이다. |

## F 투표 (실현 가능성) -- 3표

| 솔루션 | 투표수 | 투표 근거 |
|--------|:------:|----------|
| S04 출발 전 브리핑 | 1표 | 구현에 필요한 핵심 요소가 모두 성숙 기술이다. 날씨 API(OpenWeather, Google Weather), 영업시간 API(Google Places), Push 알림 인프라(Firebase Cloud Messaging) 모두 상용 서비스로 안정적으로 운영 중이다. 규제 측면에서도 위치정보법 등록제 전환(2022년)으로 진입 장벽이 낮아졌고, 개인정보보호법상 Push 알림에 필요한 동의 프로세스도 표준화되어 있다. MVP 2~3주 내 핵심 기능 구현이 현실적이다. 여행업 등록 없이 "정보 제공" 서비스로 시작할 수 있어 관광진흥법 리스크도 낮다. |
| S05 실시간 상태 배지 | 1표 | 19개 솔루션 중 가장 낮은 기술 난이도를 가진다. Google Places API의 영업 상태 조회 + 날씨 API의 현재 날씨 조회 + 단순 아이콘 렌더링만으로 MVP 핵심 기능이 완성된다. 외부 API 의존도가 높지만, 초기 5개 도시(도쿄, 오사카, 방콕, 다낭, 파리) 한정이므로 API 비용도 관리 가능하다. 4명이 독립 발상한 만큼 "일정표에 상태 아이콘이 보인다"는 UX가 직관적이어서 사용자 학습 비용이 제로에 가깝다. 데이터 정확도 90% 전제 조건 달성이 도전적이나, "회색(미확인)" 상태를 포함하여 "데이터 부재도 정보"로 설계하면 정확도 문제를 투명하게 관리할 수 있다. |
| S12 날씨 기반 일정 자동 전환 | 1표 | 날씨 API 한 개와 장소의 실내/실외 태그 하나만으로 구현 가능한 가장 단순한 자동화 솔루션이다. 날씨 예보 API의 정확도는 이미 85% 이상으로 검증되어 있고, 실내/실외 구분은 장소 등록 시 태그 하나로 해결된다. "비 오면 실내 먼저, 갠 후 실외"라는 로직은 패키지 투어 가이드가 현장에서 매일 하는 판단과 동일하다. 이 업계 관행을 자동화한 것이므로 비즈니스 로직의 검증이 이미 완료된 셈이다. 규제 리스크도 전무하며, 사용자에게 "제안" 형태로 제시하므로 추천 실패 시의 법적 면책도 용이하다. |

## 투표 총평

도메인 전문가로서 나는 "여행 산업에서 이미 검증된 가치를 기술로 자동화하는 솔루션"에 가장 높은 점수를 부여했다. 패키지 투어 가이드가 수십 년간 해온 일 -- 출발 전 브리핑, 상태 확인, 날씨에 따른 일정 조정, 맥락에 맞는 대안 제시 -- 을 자유여행에서도 AI가 대신하는 것이 이 MVP의 본질이다. S04(출발 전 브리핑)에 B표 2표와 F표 1표를 집중한 이유는, 이것이 킹핀 문제 B(사후 인지)를 가장 직접적으로 해결하면서도 구현 난이도가 낮아 MVP에서 즉시 비즈니스 가치를 증명할 수 있기 때문이다. S01(실시간 모니터링)이나 S03(AI 자동 재조정)은 궁극적 비전으로는 강력하지만, MVP 단계에서 데이터 정확도 90%를 달성하기 어렵고 대규모 인프라 투자가 필요하여 "빠른 시장 검증"이라는 MVP 목적에 부합하지 않는다. During-trip 카테고리 선점의 시간 경쟁에서는 "완벽한 큰 것"보다 "검증된 작은 것"이 먼저다.
