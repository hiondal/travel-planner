|--------|:------:|----------|

| 솔루션 | 투표수 | 투표 근거 |
|--------|:------:|----------|

### 투표 총평

DevOps 관점에서 솔루션 선택의 핵심 기준은 "배포할 수 있는가"가 아니라 "안정적으로 반복 배포할 수 있는가"다. B 투표에서는 인프라 복잡도와 비즈니스 가치의 교차점에 집중했다. S01은 이벤트 드리븐 알림 파이프라인으로, Kafka 또는 SQS 기반 메시지 큐와 푸시 서비스(FCM/APNs) 조합으로 자동화 배포가 가능하며 전체 플랫폼의 데이터 척추 역할을 한다. S03은 S01 데이터를 소비하는 서비스로 MSA 분리가 자연스러워 독립 배포가 가능하다. F 투표에서는 배포 단위가 작고 외부 의존성이 적은 솔루션에 집중했다. S04는 정적 데이터 집계 기반으로 외부 장애 전파가 최소화되며, 첫 스프린트에서 CI/CD 파이프라인을 완전 자동화할 수 있는 현실적인 출발점이다. S05는 프론트엔드 상태 표시 레이어만 추가하면 되므로 배포 리스크가 사실상 제로다. S03·S09·S10·S18처럼 초대규모 또는 AI 의존성이 높은 솔루션은 외부 API SLA 의존도가 높아 파이프라인 안정성 확보가 어렵고, 롤백 시나리오가 복잡해져 단독으로는 실현 가능성 투표를 줄 수 없었다.
