| 1 | 배달앱 | **배달의민족 실시간 배달 트래킹** | 주문 후 각 단계(접수→조리→픽업→배달중→도착)를 Push로 선제 통지. 사용자가 묻기 전에 시스템이 먼저 알려준다. 예상 도착 시간이 바뀌면 즉시 재통지. | 여행자도 출발 전에 "지금 가면 어떤 상태인지"를 Push로 먼저 받아야 한다. 목적지 상태(휴업/혼잡/정상)를 배달 상태처럼 단계별로 알려주면 도착 후 당황이 사라진다. |
| 2 | 모빌리티 | **카카오T 실시간 교통 우회 경고** | 내비게이션이 목적지가 아니라 경로 상의 교통 변화를 감지하여 사용자가 선택하기 전에 "지금 이 길은 막혀요. 우회할까요?"라고 먼저 묻는다. 운전자의 인지 부담을 시스템이 흡수. | 여행 일정도 경로다. 다음 목적지까지의 "여행 경로 상태"를 모니터링하고, 문제 감지 시 이동 결정 전에 개입하면 된다. "지금 가면 혼잡해요. 다음 장소 먼저 갈까요?" |
| 3 | 이커머스 | **쿠팡 재고 소진 임박 알림 + 대안 추천** | 관심 상품의 재고가 임박하면 자동 알림을 보내고, 품절 시 유사 상품을 즉시 대안으로 제시한다. 사용자가 "장바구니에서 사라진 것"을 발견하는 게 아니라, 미리 알고 의사결정할 수 있도록 설계. | 여행 목적지도 "소진"된다. 임시 휴업, 조기 마감, 예약 마감이 모두 "재고 소진"과 같다. 소진 전에 알리고, 소진 시 대안을 즉시 제시하는 구조. |
| 4 | 핀테크 | **토스 이상 거래 실시간 감지 알림** | 사용자의 패턴에서 벗어나는 거래를 즉시 감지하고 "이 거래 맞나요?"라고 확인을 요청. 사용자가 나중에 통장을 확인해서 알게 되는 게 아니라 거래 시점에 즉시 개입. 신뢰의 근거가 "사후 통보"가 아닌 "실시간 보호". | 여행 일정의 이상 상황도 "이상 거래"처럼 즉시 감지해야 한다. "예정 방문지 상태가 바뀌었어요. 확인할까요?" — 여행자의 일정 패턴을 알고, 그 맥락에서 이탈이 발생할 때 먼저 개입. |
| 5 | 헬스케어 | **애플워치 심장 이상 감지 + 의사 연결** | 사용자가 심장 이상을 느끼기 전에 기기가 먼저 감지하고, 위급 시 자동으로 119 연결을 제안. "모르고 있었는데 알려줬다"는 신뢰 경험이 제품의 핵심 가치. | 여행자도 "내가 몰랐는데 알려줬다"는 경험이 핵심이다. 구글맵이나 블로그에서는 능동적으로 찾아야 하지만, 우리는 여행자의 일정을 알고 있으므로 먼저 감지하고 먼저 알릴 수 있다. |

## 아이디어

| # | 구분 | 아이디어 제목 | SCAMPER/S&S 관점 | 아이디어 상세 설명 |
|---|------|------------|----------------|-----------------|
| 1 | Big Idea | **여행 일정 구독형 상태 모니터링 — "내 일정이 살아있다"** | S&S: 배달앱 트래킹 × 핀테크 이상 감지 / SCAMPER-Substitute: "내가 찾는" Pull을 "시스템이 지키는" Push로 대체 | 여행 일정에 등록된 모든 장소를 사용자가 여행하는 동안 백그라운드에서 24시간 모니터링한다. 단순히 출발 직전 한 번 확인하는 것이 아니라, 여행 기간 내내 "구독"처럼 지속 감시한다. 마치 택배 배송 상태를 트래킹하듯, 내 여행 일정 속 각 목적지의 상태(영업 중 / 임시 휴업 / 혼잡 / 정상)가 변경될 때마다 실시간으로 Push 알림이 온다. 중요한 건 "변경 발생 즉시"가 아니라 "내가 이동하기 전 적절한 시점"에 알려준다는 점이다. 배달앱이 "20분 후 도착"을 미리 알려주듯, "지금 출발하면 30분 뒤 도착할 곳이 임시 휴업 중입니다"라고 이동 결정 전에 개입한다. 이 구조는 사용자가 아무것도 하지 않아도 일정이 "살아서" 자신을 지켜보는 느낌을 준다. "일정을 짜 놓으면 시스템이 대신 감시해준다"는 경험 자체가 During-trip의 핵심 가치 제안이 된다. 정보 정확도 90% 이상이라는 전제 조건이 충족될 때, 이 구독 경험은 여행자에게 "보험"과 같은 안도감을 제공한다. |
| 2 | Big Idea | **맥락 인식 선제 대안 스냅 — "이미 다음 장소가 준비되어 있다"** | S&S: 카카오T 우회 제안 × 쿠팡 대안 추천 / SCAMPER-Adapt: 내비게이션의 실시간 경로 재탐색을 여행 일정 재조정에 적용 | 내 현재 위치, 현재 시간, 날씨, 이동 수단, 그리고 오늘 남은 일정을 동시에 이해하는 맥락 엔진이 있다고 상상해보자. 이 엔진은 목적지 이상을 감지하는 즉시, "지금 당신의 상황에서 실제로 갈 수 있는" 대안 2~3개를 미리 준비해 놓는다. 내가 알림을 열었을 때 이미 대안이 화면에 펼쳐져 있다. 23.5분을 검색하는 것이 아니라, 알림 자체가 대안 스냅이다. 중요한 것은 "대안 목록"이 아니라 "지금 내가 실제로 선택 가능한 대안"이라는 점이다. 걸어서 10분 거리, 현재 영업 중, 비슷한 카테고리, 오늘 내 동선과 연결 가능한 조건을 모두 충족한 것만 보여준다. 카카오T가 "막히는 길 대신 이 길"을 제시하듯, "닫힌 장소 대신 이 장소"를 즉시 제시한다. 사용자 입장에서는 "검색하러 앱을 열었는데 이미 답이 있다"는 경험이다. 이것이 검색 노동 23.5분을 3분 이하로 줄이는 실질적 메커니즘이다. 더 나아가 대안을 선택하면 남은 일정 전체가 자동으로 재배열되어 "연쇄 붕괴"까지 한 번에 예방한다. |
| 3 | Big Idea | **여행 상태 공개 API — "여행 경험을 플랫폼으로"** | SCAMPER-Put to other uses: 내부 모니터링 데이터를 외부 생태계에 개방 / S&S: 카카오맵 API 생태계 모델 | 우리 서비스가 수집하는 실시간 POI 상태 데이터(영업 여부, 혼잡도, 임시 이벤트)를 외부 여행 앱, 가이드북 앱, 항공사 앱에 API로 제공하는 플랫폼 전략이다. 백엔드 개발자로서 이 아이디어를 꺼내는 이유는 기술 때문이 아니라 "여행자로서" 다양한 앱에서 동일한 신뢰 정보를 만나고 싶기 때문이다. 지금은 구글맵, 네이버, 트립어드바이저가 각자 다른 정보를 가지고 있어 어느 것을 믿어야 할지 모른다. 우리가 "가장 정확한 During-trip 상태 정보"를 갖게 된다면, 그 데이터를 독점하는 것보다 생태계에 개방하는 것이 사용자에게 더 큰 가치다. 여행자는 어떤 앱을 쓰든 동일하게 정확한 정보를 받게 된다. 회사 입장에서는 B2B API 수익 모델이 열리고, 더 많은 앱에서 우리 데이터를 쓸수록 데이터 정확도가 높아지는 플라이휠이 형성된다. "우리 앱만 쓰세요"가 아니라 "어디서든 우리 데이터를 만나세요"라는 접근이 During-trip 카테고리 선점의 더 빠른 경로가 될 수 있다. 단기적으로는 가장 어렵지만, 장기적으로 가장 강력한 해자를 만든다. |
| 4 | Little Win | **출발 전 30초 브리핑 카드 — "오늘 여행 날씨예보"** | SCAMPER-Eliminate: 복잡한 알림 설정 과정 제거, 매일 아침 자동 배달 / S&S: 날씨앱의 "오늘 날씨" 카드 UX 모델 | 매일 아침 첫 일정 출발 30분 전, 오늘 계획된 장소들의 상태를 한 장의 카드로 요약해서 보내주는 기능이다. 복잡한 실시간 알림 없이도, 하루를 시작할 때 "오늘 일정 이상 없음" 또는 "3번째 장소 주의 필요"를 단 한 번의 확인으로 파악할 수 있다. 날씨앱이 "오늘 오후 비"를 아침에 알려주듯, 여행 일정앱이 "오늘 4번째 장소 임시 휴업 가능성"을 아침에 알려준다. 이 기능의 핵심 가치는 단순함이다. 실시간 Push를 켜두기 부담스러운 사용자도, 아침 브리핑 하나로 하루 여행의 리스크를 사전에 파악한다. 기술적으로 가장 가볍고, 사용자 진입 장벽이 가장 낮다. "MVP의 MVP"로서, 이 기능 하나만으로도 킹핀 문제(사후 인지)의 80%를 해소할 수 있다. 사용자가 앱을 쓰는 습관을 만드는 가장 자연스러운 진입점이기도 하다. |
| 5 | Little Win | **여행 버디 공유 링크 — "우리 오늘 어디가?"** | SCAMPER-Combine: 일정 공유 + 실시간 상태 + 그룹 투표 결합 / S&S: 구글 독스의 실시간 공동 편집 + 카카오톡 공유 | 솔로 여행자가 현지에서 잠깐 만난 여행자, 혹은 같은 숙소 게스트와 실시간으로 오늘 일정을 공유할 수 있는 임시 링크를 생성하는 기능이다. "오늘 오후 같이 돌아다닐래요?"라고 말했을 때, 링크 하나로 내 오늘 일정과 각 장소의 현재 상태를 공유하고, 상대방이 "이건 가고, 저건 패스"를 눌러 즉석 합의를 이끌어낸다. 솔로 여행자이지만 현지에서 동행이 생기는 순간을 위한 기능이다. 최우선 세그먼트인 솔로 트래블러(25~34세)는 SNS에 익숙하고, 공유 링크 하나로 즉흥 그룹 여행이 가능해진다. 더불어 이 공유 링크가 바이럴 성장의 채널이 된다. 기능 자체는 단순(링크 생성, 상태 조회, 투표 집계)하지만 경험은 풍부하다. 그룹 여행자를 위한 본격 기능을 만들기 전, 가장 가벼운 형태로 "그룹 경험"을 MVP에 포함하는 방법이다. |
| 6 | Crazy Idea | **여행 상태 감지 자동화 봇 — "내가 직접 가서 확인해 드릴게요"** | SCAMPER-Reverse: 사용자가 현장에 가는 것이 아니라 시스템이 "대신 가서" 확인하는 구조 역전 / S&S: 미스터리 쇼퍼 + 크라우드소싱 배달 모델 | 현지에 있는 여행자들을 "여행 상태 제보 에이전트"로 활용하여, 앱 사용자가 목적지에 도착할 때마다 자동으로 해당 장소의 현재 상태(영업 중/혼잡도/특이사항)를 체크인하게 하는 구조다. 단순 리뷰가 아니라 "지금 이 순간 상태"를 10초 이내로 입력하는 체크인이다. 체크인 1회당 포인트가 쌓이고, 이 포인트로 다음 여행 구독료를 할인받는다. 미쳐보이는 이유는 이것이다. 우리 서비스의 가장 큰 기술적 약점인 "실시간 정보 정확도 90% 달성"을, AI나 공식 API가 아니라 여행자 커뮤니티가 해결하게 하는 발상이다. 구글맵의 리뷰 시스템이 수백만 명의 자발적 기여로 유지되듯, 우리의 실시간 상태 정보도 여행자들이 서로 위해 서로 만들어간다. 미스터리 쇼퍼를 크라우드소싱한 것과 같다. 정확도 문제를 데이터 문제가 아닌 커뮤니티 문제로 재정의하면, 가장 어려운 전제 조건이 동시에 가장 강력한 성장 엔진이 된다. 여행자들이 서로의 여행을 지켜주는 생태계 — 기술이 아니라 사람이 During-trip 공백을 채운다. |
