### 2-2. 인과관계 근거 (인터뷰 데이터 기반)

**B -> A 인과관계: "사후 인지가 검색 노동을 강제한다"**

| 근거 | 출처 | 원문 |
|------|------|------|
| B 해결 시 A 발생 자체가 감소 | S1 (솔로) | "미리 알았으면 검색 자체를 안 해도 되니까." |
| B의 감정적 파급이 A의 탐색 품질을 저하 | S2 (그룹) | "한 번의 사후 인지가 남은 여행 전체의 톤을 바꿔버릴 수 있다." |
| B가 전체 문제 체인의 시작점 | S3 (초보) | "B가 문제 체인의 트리거 역할을 한다." |

- B(사후 인지)가 발생하지 않으면, 여행자는 이동 전에 상황 변화를 인지하고 여유 있게 대안을 선택할 수 있다. 즉, A(대안 탐색)의 발생 빈도와 긴급도가 동시에 감소한다.
- B가 발생한 상태에서의 A는 "당황 + 시간 압박 + 감정 저하" 속에서의 탐색이므로, 탐색 시간이 길어지고 탐색 품질이 저하된다.

**A -> C 인과관계: "대안 탐색의 결과가 일정 전체를 흔든다"**

- A에서 선택한 대안 장소는 원래 일정과 동선/시간/예약이 맞지 않아 후속 일정 2~3곳의 연쇄 변경을 유발한다.
- 3명 모두 1개 장소 변경 시 평균 2~3곳의 추가 변경이 발생했다고 증언.

**B -> C 직접 의존관계: "감지 없이 재조정은 무의미"**

- C의 첫 번째 복합 근본원인이 "B(감지/알림)와 연계된 선제적 재조정 파이프라인의 부재"다.
- S1: "감지 없이 재최적화만 있으면 소용없다" -- C의 해결은 B의 해결을 전제 조건으로 한다.

### 2-3. 인과 구조의 의미

```
B(예방) --> A(대응) --> C(복구)

B를 해결하면:
  - A의 발생 빈도 자체가 감소 (사전 인지 -> 사전 대응 가능)
  - C의 트리거 빈도도 감소 (사전 조정 -> 연쇄 붕괴 방지)

A만 해결하면:
  - B는 여전히 발생 (도착 후 인지는 그대로)
  - C도 여전히 발생 (급한 대안 선택 -> 동선 불일치 -> 연쇄 변경)

C만 해결하면:
  - B와 A는 여전히 발생
  - 감지(B) 없이 재조정(C)만 있으면 소용없음 (S1 지적)
```

3명의 인터뷰이가 모두 이 인과 구조를 독립적으로 인식했다는 점이 핵심이다. 이것은 연구자의 해석이 아니라 사용자 스스로가 경험한 구조다.

## 3. 5가지 기준 평가 (각 5점)

### 3-1. 평가 기준 정의

| 기준 | 핵심 질문 | 평가 포인트 |
|------|---------|-----------|
| 영향력 (Impact) | 해결 시 다른 문제도 함께 해결되는가? | 연쇄 해결 문제의 수와 중요도 |
| 빈도 (Frequency) | 고객이 얼마나 자주 겪는가? | 여행 중 발생 빈도, 세그먼트별 보편성 |
| 심각도 (Severity) | 고객에게 미치는 고통의 정도는? | 시간 손실, 비용 손실, 감정적 고통 |
| 근본성 (Root Cause) | 다른 문제들의 원인이 되는가? | 인과 체인에서의 위치, 트리거 여부 |
| 해결가능성 (Feasibility) | 현실적으로 해결 가능한가? | 기술 성숙도, 자원, 시간, 비용 |

### 3-3. 문제가설 A: 대안 탐색 시간 소모 -- 평가

| 기준 | 점수 | 근거 |
|------|:----:|------|
| 영향력 | **3** | A 해결 시 탐색 시간은 단축되나, B(사후 인지) 자체는 해결되지 않음. C(연쇄 붕괴)도 부분 완화에 그침(더 나은 대안 선택으로 연쇄 영향 감소 가능하나, 연쇄 구조 자체는 남음). A는 B의 결과이지 원인이 아니므로 역방향 해결력이 없음. |
| 빈도 | **5** | 인터뷰 3명 전원이 복수 경험. 돌발 상황 발생 시 100% 수반됨. 체감 시간 30분 내외(기존 가설 23.5분보다 체감이 더 김). 그룹 여행 시 합의 비용 포함하면 40~60분+. 세그먼트 무관 보편적 (중요도 5.0). |
| 심각도 | **4** | 시간 손실 23.5분+(그룹 40~60분+). 검색 피로로 이후 경험 품질까지 저하. 그러나 감정적 고통의 시작점은 A가 아니라 B(사후 인지)임. A는 B에 의해 이미 저하된 감정 상태(1.6~1.9)에서의 추가 부담. 불편도 4.3 (B의 5.0보다 낮음, S1/S2는 4점). |
| 근본성 | **2** | A는 B의 결과로 발생하는 문제이지, 다른 문제의 원인이 되는 것은 아님. A->C로의 영향은 있으나, 이는 A 자체의 근본성이라기보다 B->A->C 체인의 중간 단계. 5WHY 분석에서 Why5("여행 중 실시간 대안 추천 카테고리 자체가 정의되지 않음")는 B의 Why5("관광 데이터 생태계 구조")와 독립적이나, 인과 체인상 B 이후에 위치. |

| **합계** | **18/25** | |

### 3-4. 문제가설 C: 연쇄 일정 붕괴 -- 평가

| 기준 | 점수 | 근거 |
|------|:----:|------|
| 영향력 | **2** | C 해결이 B나 A를 해결하지 못함. C는 인과 체인의 말단에 위치하여 역방향 해결력이 없음. 오히려 C의 해결은 B의 해결을 전제 조건으로 함(S1 "감지 없이 재최적화만 있으면 소용없다"). |
| 빈도 | **4** | 일정 변경 1건당 2~3곳 연쇄 영향(3명 전원 경험). 그러나 B(사후 인지)보다 발생 빈도가 낮음 -- 모든 사후 인지가 연쇄 붕괴로 이어지지는 않음. 중요도 4.3 (B, A보다 낮음). 빡빡한 일정을 짜는 사람일수록 발생 빈도 높음(S1). |
| 심각도 | **4** | 감정적으로 가장 취약한 시점(1.6~1.9)에 인지적으로 가장 복잡한 작업(수동 재계산) 수행 -- "감정-인지 미스매치". 손실 회피(loss aversion) 심리로 "포기해야 하는 장소"에 대한 고통. 그러나 불편도 4.0 (B의 5.0, A의 4.3보다 낮음). S3(초보)에게 더 심각하나, 최우선 세그먼트(S1 솔로)는 상대적으로 수용 경향. |

## 4. 킹핀 문제 선정

### 4-1. 선정 결과

> **킹핀 문제: B. 사후 인지**
>
> "여행자의 80%가 임시 휴업, 혼잡, 교통 중단 등의 돌발 상황을 목적지에 도착한 후에야 인지하여 이동 시간이 매몰 비용이 되고 감정 곡선이 급락하는 문제"

### 4-2. 선정 근거

**1. 5가지 기준 평가에서 압도적 1위 (24/25점)**

- 영향력, 빈도, 심각도, 근본성 4개 기준에서 만점(5점).
- 2위 A(18점)와 6점 차이, 3위 C(14점)와 10점 차이.
- 해결가능성만 4점인 이유는 데이터 정확도 90% 달성이라는 기술적 도전이 있기 때문이나, 이는 불가능이 아닌 도전적 과제 수준.

**2. 인터뷰 검증 데이터에서 4관왕**

| 지표 | B. 사후 인지 | A. 대안 탐색 | C. 연쇄 붕괴 |
|------|:----------:|:----------:|:----------:|
| 중요도 | 5.0 (만점) | 5.0 (만점) | 4.3 |
| 불편도 | **5.0 (만점)** | 4.3 | 4.0 |
| 동의율 | **100%** | 100% | 66.7% |
| 최심각 투표 | **3/3 만장일치** | 0/3 | 0/3 |

B는 3개 문제 중 유일하게 모든 정량 지표에서 최고점을 기록했다. 특히 "가장 심각한 문제" 투표에서 3명 전원이 B를 선택한 것은 통계적 확률로 1/27(3.7%)에 해당하는 극히 낮은 우연 확률이다.

**3. 인과 체인의 최상위 트리거**

B는 B->A->C 오류 연쇄의 첫 번째 고리다. 볼링의 5번 핀처럼, B를 쓰러뜨리면 A와 C가 연쇄적으로 완화된다.

- B 해결 -> A의 발생 빈도 감소 (사전 인지 -> 사전 대응 가능)
- B 해결 -> C의 트리거 빈도 감소 (사전 조정 -> 연쇄 붕괴 방지)
- C의 첫 번째 근본원인이 "B와 연계된 파이프라인의 부재"이므로, B 없이 C를 해결하는 것은 구조적으로 불가능

**4. 감정 보호가 시간 절약보다 강력한 사용자 동기**

피보팅 과정에서 발견된 핵심 인사이트다. 인터뷰 3명 전원이 "시간 절약보다 감정 보호가 더 중요하다"고 독립적으로 확인했다. B의 핵심 가치는 "감정 절벽 방지"이며, 이것이 리텐션과 유료 전환의 진짜 트리거다. A의 "23.5분을 5분으로"라는 시간 절약 가치는 B가 해결된 상태에서 더 큰 의미를 가진다.

### 4-3. 킹핀 해결 시 연쇄 해결 메커니즘

```
[킹핀 해결: B. 사후 인지]
"이동 전에 돌발 상황을 선제적으로 알려준다"
          │
          ├──> [A. 대안 탐색 시간 소모 -- 연쇄 완화]
          │    - 사전 인지 상태에서 여유 있게 대안 탐색 가능
          │    - 감정이 안정된 상태(3.6 유지)에서 탐색 -> 탐색 품질 향상
          │    - "미리 알았으면 검색 자체를 안 해도 되니까" (S1)
          │    - A의 발생 빈도 자체가 감소
          │
          └──> [C. 연쇄 일정 붕괴 -- 연쇄 완화]
               - 사전 조정으로 연쇄 붕괴 방지 가능
               - B->C 파이프라인이 구축되어 감지 즉시 재조정 트리거
               - 감정-인지 미스매치 완화 (여유 있는 상태에서 재조정)
               - 연쇄 변경 횟수 자체가 감소
```

**구체적 시나리오로 설명하면:**

현재 (B 미해결):
1. 여행자가 30분 이동하여 목적지 도착 -> 임시 휴업 발견 (B 발생)
2. 감정 곡선 3.6->1.6 급락, 30분 이동 시간 매몰
3. 당황한 상태에서 30분간 대안 검색 (A 발생)
4. 급히 선택한 대안이 기존 동선과 불일치 -> 후속 2~3곳 연쇄 변경 (C 발생)

킹핀 해결 후 (B 해결):
1. 이동 전에 "목적지 임시 휴업" 알림 수신 (B 해결)
2. 감정 곡선 3.6 유지, 이동 시간 매몰 없음
3. 여유 있는 상태에서 대안 2~3개 중 선택 (A 완화 -- 발생하더라도 3~5분으로 단축)
4. 사전 선택이므로 동선 일치 대안 가능 -> 연쇄 변경 최소화 (C 완화)

### 4-4. 킹핀 문제의 핵심 근본원인

**"여행 일정 기반 선제적 리스크 감지/알림(Push) 시스템의 부재"**

현재 모든 여행 도구는 사용자가 직접 정보를 찾아야 하는 Pull 모델이다. 여행자의 일정에 등록된 장소를 실시간으로 모니터링하고, 이동 전에 상태 변화를 선제적으로 알려주는 Push 모델이 부재하기 때문에, 여행자는 매번 도착해서야 당황하게 된다.

이 근본원인이 해소되면:
- 심리적 측면: 확증 편향이 존재하더라도 시스템이 보정
- 맥락적 측면: 현지어 장벽이 있더라도 시스템이 번역/감지
- 커뮤니케이션 측면: 현장 안내문이 아닌 사전 알림으로 정보 전달 체계 전환

### 4-5. 전제 조건

킹핀 문제 해결의 전제 조건으로 다음이 반드시 충족되어야 한다:

> **정보 정확도 90% 이상 = 서비스 존재의 전제 조건**
>
> S1: "정확도 90% 이상은 보장되어야 신뢰가 쌓인다."
> 3명 전원: 구글맵/블로그의 정보 부정확성이 사후 인지의 직접 원인.
>
> 우리 서비스가 동일한 정보 오류를 반복하면 기존 도구와 차별성이 없다.
> 정확도는 기능이 아니라 존재 이유(reason to exist)다.
