## 1. Problem (문제)

### 검증된 3대 문제 (인터뷰 3명, 만장일치)

| 순위 | 문제 | 검증 강도 | 핵심 데이터 |
|:----:|------|----------|-----------|
| **1** | **B. 사후 인지**: 여행자의 80%가 임시 휴업, 혼잡, 교통 중단 등을 도착 후에야 인지. 감정 곡선 3.6->1.6 급락 (-2.0점). 이동 시간 15~75분 매몰 비용 | 중요도 5.0, 불편도 5.0, 동의율 100%, 만장일치 최심각 | 3명 전원 "감정 보호 > 시간 절약" |
| **2** | **A. 대안 탐색 시간 소모**: 대안 찾기에 23.5분+ 소요 (그룹 40~60분+). 구글맵/네이버/인스타 "삼각 동선" 반복 | 중요도 5.0, 불편도 4.3, 동의율 100% | 체감 시간 30분 내외 (가설보다 심각) |
| **3** | **C. 연쇄 일정 붕괴**: 1건 변경 시 2~3곳 연쇄 영향. 감정 취약 시점(1.6~1.9)에 수동 재계산 강제 | 중요도 4.3, 불편도 4.0, 동의율 66.7% | 부분 검증 (B 해결이 전제 조건) |

### 인과 구조

```
B(사후 인지) ──트리거──> A(대안 탐색 23.5분+) ──유발──> C(연쇄 일정 붕괴)
   "예방"                    "대응"                      "복구"
```

B를 해결하면 A의 발생 빈도 자체가 감소하고, C의 트리거 빈도도 줄어든다. 이것이 킹핀 B를 MVP 1순위로 선정한 핵심 근거다.

### 기존 대안 (Existing Alternatives)

현재 여행자들이 사용하는 도구와 한계:

| 기존 대안 | 역할 | 한계 |
|----------|------|------|
| 구글맵 | 위치/거리 검색 | 임시 휴업 갱신율 30% 미만, Pull 모델(직접 검색해야 정보 제공) |
| 네이버 블로그 | 한국어 리뷰 확인 | 2~3년 전 정보, 실시간성 없음 |
| 인스타그램 | 비주얼 확인 | 영업 상태/혼잡도 정보 없음 |
| 트리플 | 일정 관리 | "여행 전 계획" 전용, 실시간 상황 변화 대응 불가 |
| TripIt | 예약 정리 | 현지 맛집/관광지 추천 미제공, AI 재조정 없음 |

핵심 공백: "여행 중 실시간 실행"을 지원하는 통합 서비스가 시장에 부재.

## 3. Unique Value Proposition (고유 가치 제안)

### 핵심 메시지

> **"여행 중 당황하지 않는 단 하나의 앱"**

### 하위 메시지

- **감정 보호**: "도착하기 전에 알려주니까, 감정 절벽이 사라진다"
- **시간 환원**: "23.5분의 검색 노동이 3분으로. 절약된 시간은 여행으로"
- **통제감 유지**: "예상 밖 상황에서도 '괜찮아, 대안이 있어'라는 안심감"

### UVP의 3가지 축

| 축 | 기존 도구 | 우리 서비스 |
|----|----------|-----------|
| **시점** | 여행 "전" 계획 | 여행 "중" 실행 |
| **방식** | Pull (직접 검색) | Push (선제적 알림) + Pull (배지 확인) |
| **범위** | 정보 나열 | 맥락 기반 대안 제시 + 일정 반영 |

### High-Level Concept

"여행 중의 웨이즈(Waze)" -- 운전 중 실시간 교통 정보를 제공하는 Waze처럼, 여행 중 실시간 상황 변화를 감지하고 최적 경로를 안내한다.

## 4. Solution (솔루션)

### 핵심 솔루션 3개 -- 통합 UX 흐름

```
[일정표 화면]
    |
    +-- S05 상태 배지: 초록/노랑/빨강/회색 -------- Pull 기반 인지
    |
    +-- 출발 15~30분 전
    |   +-- S04 출발 전 브리핑: Push 알림 -------- 사전 인지
    |       +-- 정상 -> "안심 브리핑" (습관 형성)
    |       +-- 주의/위험 -> 브리핑 + 대안 카드 연결
    |
    +-- 돌발 상황 발생 시
        +-- S06 대안 카드: 카드 3장 즉시 제시 ---- 검색 노동 제거
            +-- 탭 -> 일정 반영 완료
```

| 솔루션 | 핵심 기능 | 킹핀 연결 | MVP Sprint |
|--------|----------|----------|:----------:|
| **S05 실시간 상태 배지** | 일정표 장소별 초록/노랑/빨강/회색 상태 아이콘 | B(사후 인지) 시각적 해결 | Sprint 1 (1~2주) |
| **S04 출발 전 브리핑** | 출발 15~30분 전 Push 알림, 영업/혼잡/날씨/이동시간/총평 5가지 | B(사후 인지) 직접 해결 | Sprint 2 (2~3주) |
| **S06 맥락 맞춤 대안 카드** | 돌발 시 대안 2~3개 카드 즉시 제시, 탭으로 일정 반영 | A(대안 탐색) 직접 해결 | Sprint 3 (2~3주) |

### MVP 범위 (MoSCoW)

| 분류 | 기능 | 근거 |
|------|------|------|
| **Must Have** | S05 상태 배지, S04 출발 전 브리핑, S06 대안 카드, 일정 등록/관리, Push 알림 | 킹핀 B+A 해결에 필수 |
| **Should Have** | 안심 브리핑(정상 시), 대안 채택 후 일정 자동 반영, 오프라인 캐싱 | 리텐션/UX 완결성 |
| **Could Have** | 그룹 공유 링크, 다국어 지원, 혼잡도 시간대별 그래프 | 확장 세그먼트 대응 |
| **Won't Have (MVP)** | AI 자동 일정 재조정(S03), 현지 친구 컨시어지(S09), 감정 센서 | 기술 성숙도/범위 초과 |

## 6. Revenue Streams (수익 구조)

### 수익 모델 설계 원칙

1. **여행 건당 과금이 1차 수익**: 인터뷰 WTP 5,000~10,000원/여행건 검증 완료. 구독보다 진입 장벽이 낮고, 여행 빈도와 과금 시점이 일치.
2. **프리미엄 구독이 2차 수익**: 연 3회 이상 여행자에게 구독이 더 합리적인 가격 구조 설계.
3. **무료 티어로 습관 형성**: 무료 사용자에게도 배지(S05) + 제한된 브리핑을 제공하여 가치 체험 후 전환.

### 가격 전략

| 티어 | 가격 | 포함 기능 | 타겟 |
|------|------|----------|------|
| **Free** | 0원 | 상태 배지(S05), 1일 1회 브리핑(S04), 도시 1개 | 신규 사용자 (습관 형성) |
| **Trip Pass** (건당) | 7,900원/여행건 | 브리핑 무제한(S04), 대안 카드(S06), 전체 도시, 오프라인 캐싱 | 연 1~2회 여행자 |
| **Pro** (월 구독) | 4,900원/월 | Trip Pass 전체 + 안심 브리핑 상세 리포트, 우선 알림, 히스토리 | 연 3회+ 여행자, 리피터 |
| **Pro Annual** (연간) | 39,900원/년 (월 3,325원) | Pro 동일 + 2개월 무료 혜택 | 헤비 트래블러 |

### 가격 설정 근거

- **Trip Pass 7,900원**: WTP 중간값(5,000~10,000원)에서 심리적 저항이 낮은 7,900원 설정. 여행 1회 총 예산 100~200만원 대비 0.4~0.8%로 미미한 수준. "23.5분 절약 = 시급 환산 약 20,000원" 대비 합리적.
- **Pro 4,900원/월**: 연 3회 여행 시 Trip Pass 23,700원 > Pro 58,800원이 아닌, Pro 58,800원 < Trip Pass 3회 23,700원이 되도록 설계. 연 3회 기준 Trip Pass 합계 23,700원 vs Pro Annual 39,900원이므로, 연 5회 이상에서 구독이 유리(39,500원 vs 39,900원). 실질적으로 연 4회 이상 여행자에게 구독 전환 인센티브 제공.

### LTV 산출

| 시나리오 | 가정 | LTV |

## 9. Unfair Advantage (경쟁 방어벽)

### 주의: 일반적 기능은 Unfair Advantage가 아니다

Push 알림, 상태 배지, 대안 추천 등은 기능(Feature)이지 경쟁 방어벽이 아니다. 경쟁사가 6개월 내에 복제할 수 있는 것은 Unfair Advantage에 해당하지 않는다. 진정한 Unfair Advantage는 시간이 지날수록 강화되며, 쉽게 복제할 수 없는 구조적 우위다.

### Unfair Advantage 3가지

**1. During-trip 카테고리 최초 정의 및 선점 (Category Creation)**

- 기존 OTA/여행앱이 모두 Before-trip에 집중하는 가운데, "여행 중 실시간 일정 최적화"라는 카테고리 자체를 최초로 정의한다.
- 카테고리 창출자(Category Creator)는 해당 카테고리에서 70% 이상의 마인드셰어를 장기 보유하는 경향이 있다 (HBR, Category Design 연구).
- Google Trips 종료(2019년) 이후 이 영역의 공백이 7년간 지속되어, 선점의 시간 창이 열려 있다.
- 경쟁사가 진입하더라도 "OO = 여행 중 가이드"라는 카테고리 연상을 먼저 확보한 사업자가 유리하다.

**2. B->A->C 인과 체인 기반 누적 행동 데이터 자산 (Data Moat)**

- 사용자가 브리핑(B 해결)을 받고, 대안 카드(A 해결)를 선택하고, 일정을 반영(C 부분 해결)하는 전 과정의 행동 데이터가 누적된다.
- "어떤 상황에서 어떤 대안을 선택했고, 만족했는가"라는 데이터는 현재 어떤 서비스도 보유하지 않은 신규 데이터 자산이다.
- 이 데이터가 축적될수록 대안 추천의 정확도가 향상되고(데이터 플라이휠), 후발 주자는 이 데이터 격차를 단기간에 메울 수 없다.
- 가명처리 후 AI 학습에 활용 가능(개인정보보호법 제28조의2).

**3. 도시별 실시간 POI 정확도 데이터 플라이휠 (Accuracy Flywheel)**

- 사용자의 피드백("이 장소 실제로 영업 중이었다/아니었다")이 POI 정확도를 지속적으로 교정한다.
- 정확도 향상 -> 사용자 신뢰 증가 -> 더 많은 사용 -> 더 많은 피드백 -> 정확도 추가 향상의 선순환.
- S1(솔로 트래블러)이 명시적으로 "정확도 90% 이상이 신뢰의 최소 기준선"이라고 발언한 만큼, 이 플라이휠은 서비스 존재의 전제 조건이자 가장 강력한 방어벽이다.
- 도시별로 축적되므로, 서비스 도시가 늘어날수록 경쟁사의 진입 비용이 기하급수적으로 증가한다.

## 15. 총평

PO로서 이 비즈니스 모델의 핵심 판단은 세 가지다.

**첫째, "왜 지금인가"에 대한 답이 명확하다.** During-trip 카테고리는 Google Trips 종료(2019년) 이후 7년간 구조적 공백 상태이며, AI 기술(LLM, 실시간 데이터 통합)이 최근에야 이 문제를 해결할 수 있는 수준에 도달했다. AI 관광 시장 CAGR 26.7%, 솔로 여행 시장 CAGR 14.3%라는 성장 동인이 우리의 타이밍과 정확히 맞는다.

**둘째, "누가 돈을 내는가"가 검증되었다.** 인터뷰 3명 전원이 WTP 5,000~10,000원/여행건을 확인했고, 이 WTP의 근거가 "감정 절벽 방지"와 "23.5분 시간 환원"이라는 두 가지 검증된 고객 가치에 기반한다. Trip Pass 7,900원이라는 가격은 여행 예산의 0.4~0.8%에 불과하며, 심리적 저항이 낮다.

**셋째, "언제 흑자인가"가 현실적이다.** 월간 BEP는 유료 사용자 2,800명, 누적 BEP는 출시 후 약 30개월(3년차 2분기)이다. 3년차 영업이익률 46%는 SaaS 기업의 건전한 수준이며, LTV/CAC 5.2x는 지속 가능한 성장을 뒷받침한다.

팀 행동원칙 "Fast fail, Learn and Pivot"에 따라, MVP 5~8주 출시로 핵심 가설(브리핑 열람률 70%, 정확도 90%, NPS 40)을 빠르게 검증하고, 데이터 기반으로 가격과 기능을 조정해 나간다. 가장 가치 있는 것이 아니라, **가장 가치를 증명할 수 있는 것부터** 만든다.
