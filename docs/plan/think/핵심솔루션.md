## 선정 결과: 3개 핵심 솔루션

### 핵심 솔루션 1: S04 출발 전 브리핑 (No Brainers)

> **"다음 목적지로 출발하기 전, 목적지 상태를 간결하게 브리핑한다."**

| 항목 | 내용 |
|------|------|
| **매트릭스 위치** | **No Brainers** (유일) |
| **득표** | B 7표 (5명 투표) + F 14표 (9명 전원 투표) = **21표** |
| **킹핀 연결** | B(사후 인지) 직접 해결 — "출발 전에 미리 알려준다" |
| **구현 규모** | 소규모 (기존 API 조합 + 스케줄링) |
| **MVP 구현 기간** | 2~3주 |

**선정 근거**:
- 19개 솔루션 중 **유일한 No Brainer**: 비즈니스 가치(B=7)와 실현 가능성(F=14) 모두 높은 유일한 솔루션
- **9명 전원 F표 투표**: 팀 전원이 "MVP에서 만들 수 있다"에 동의
- **Needs Statement 직결**: "목적지에 도착하기 전에 미리 알 수 있는 경험" = 출발 전 브리핑
- **습관 형성**: 정상 시에도 안심 브리핑을 제공하여 "알림 = 안심 신호" 습관 형성 → 리텐션의 기반
- **수익화 연결**: 건당 과금의 첫 번째 유료 기능 전환 후보 (WTP 5,000~10,000원)
- **기술 성숙도**: 날씨 API + Places API + Push 알림 = 검증된 기술 조합

**핵심 기능**:
- 출발 15~30분 전 자동 Push 알림
- 영업상태 / 혼잡도 / 날씨 / 이동시간 / 총평 5가지 정보
- 정상 시 안심 브리핑 + 이상 시 주의 브리핑
- 주의 필요 시 대안 카드(S06) 연결

### 핵심 솔루션 2: S05 실시간 상태 배지 (Utilities → MVP 기반)

> **"일정표의 각 장소 옆에 초록/노랑/빨강/회색 상태 아이콘을 표시한다."**

| 항목 | 내용 |
|------|------|
| **매트릭스 위치** | Utilities (높은 실현 가능성) |
| **득표** | B 0표 + F 6표 (6명 투표) = **6표** |
| **킹핀 연결** | B(사후 인지) 시각적 해결 — "한눈에 상태 파악" |
| **구현 규모** | 매우 소규모 (API 상태 조회 + 아이콘 렌더링) |
| **MVP 구현 기간** | 1~2주 |

**선정 근거**:
- **MVP Sprint 1 최우선 구현**: 19개 중 가장 낮은 구현 난이도. "배포 리스크 제로"(파이프)
- **시각적 기반 인프라**: S04(브리핑), S02(대시보드), S01(모니터링) 등 모든 후속 기능의 시각 언어 기반
- **직관적 UX**: 초록/노랑/빨강 신호등 메타포 — 학습 비용 제로
- **Pull 기반 사전 인지**: Push 알림 없이도 일정표를 볼 때 자연스럽게 상태 인지 가능
- **"데이터 부재도 정보"**: 회색(미확인) 배지로 정확도 문제를 투명하게 관리

**핵심 기능**:
- 초록(정상) / 노랑(주의) / 빨강(위험) / 회색(미확인) 4단계 상태
- 일정표 내 장소명 옆 배지 표시
- 탭 시 상세 정보 팝업 → S06(대안 카드) 연결

### 핵심 솔루션 3: S06 맥락 맞춤 대안 카드 (Utilities → A 해결)

> **"돌발 상황 시 대안 2~3개를 카드 형태로 즉시 제시. 검색 과정 자체가 사라진다."**

| 항목 | 내용 |
|------|------|
| **매트릭스 위치** | Utilities (균형잡힌 양축 점수) |
| **득표** | B 3표 (3명 투표) + F 5표 (5명 투표) = **8표** |
| **킹핀 연결** | A(대안 탐색) 직접 해결 — "23.5분 → 3분 이하" |
| **구현 규모** | 소규모~중규모 (맥락 엔진 + 장소 데이터) |
| **MVP 구현 기간** | 2~3주 |

**선정 근거**:
- **A(대안 탐색) 직접 해결**: S04/S05가 B(사전 인지)를 해결하면, 남은 문제는 "대안을 어떻게 찾나"
- **즉각적 가치 체감**: "23.5분 검색 → 3분 이하" = 측정 가능한 핵심 KPI
- **효율 추구형 솔로 트래블러 최적**: "열고, 보고, 탭한다" 3단계 UX
- **점진적 고도화**: MVP에서 규칙 기반(거리+카테고리+영업 중 필터) → 이후 LLM 기반 개인화
- **S04 시너지**: 브리핑에서 "주의 필요" 시 대안 카드로 자연스러운 전환

**핵심 기능**:
- 카드 3장 = 장소명 + 거리 + 평점 + 혼잡도 + 추천이유
- 탭 한 번으로 일정에 반영
- S04(브리핑) 및 S05(배지) 위험 감지 시 자동 연결

## 3개 솔루션의 통합 UX 흐름

```
[일정표 화면]
    │
    ├─ S05 상태 배지: 각 장소별 초록/노랑/빨강/회색 ─── Pull 기반 인지
    │
    ├─ 출발 15~30분 전
    │   └─ S04 출발 전 브리핑: Push 알림 ─── 사전 인지
    │       ├─ 정상 → "안심 브리핑" (습관 형성)
    │       └─ 주의/위험 → 브리핑 + 대안 카드 연결
    │
    └─ 돌발 상황 발생 시
        └─ S06 대안 카드: 카드 3장 즉시 제시 ─── 검색 노동 제거
            └─ 탭 → 일정 반영 완료
```

### 킹핀 B→A→C 커버리지

| 킹핀 | 해결 솔루션 | 해결 방식 | 커버율 |
|------|-----------|----------|:------:|
| **B (사후 인지)** | S05(배지) + S04(브리핑) | 시각적 Pull + 선제적 Push로 사후→사전 전환 | **100%** |
| **A (대안 탐색)** | S06(대안 카드) | 맥락 기반 카드 3장으로 23.5분→3분 | **100%** |
| **C (연쇄 붕괴)** | S04+S06 조합 | 사전 인지+즉시 대안으로 연쇄 차단 | **부분** |

> C(연쇄 붕괴)의 완전 해결은 S03(AI 자동 재조정 엔진)이 필요하며, 이는 MVP 이후 로드맵에 포함한다.

## 선정 총평

PO로서 이번 선정의 핵심 철학은 **"가장 가치 있는 것이 아니라, 가장 가치를 증명할 수 있는 것부터"**이다.

S01(실시간 모니터링)이 B표 14표로 가장 높은 비즈니스 가치를 인정받았지만, F표 0은 "지금 만들 수 없다"는 팀 전원의 판단이다. 반면 S04+S05+S06은 합계 35표(전체 54표의 65%)를 받으며, "비즈니스 가치가 있고 + 만들 수 있다"는 동시 합의를 얻었다.

이 세 솔루션은 단순한 기능 나열이 아니라, **인지(Pull: 배지) → 알림(Push: 브리핑) → 행동(Action: 대안 카드)**이라는 하나의 완결된 사용자 경험 흐름을 구성한다. 킹핀 B(사후 인지)와 A(대안 탐색)를 동시에 해결하며, MVP 5~8주 내 시장 검증이 가능하다.

팀 행동원칙 "Fast fail, Learn and Pivot"에 따라, S04+S05+S06으로 빠르게 핵심 가치를 검증한 후, 시장 반응에 따라 S01(모니터링 엔진)과 S03(AI 재조정)으로 확장하는 것이 PO의 최종 판단이다.
