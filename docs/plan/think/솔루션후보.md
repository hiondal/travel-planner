| 핵심 메커니즘 | 초록(정상)/노랑(주의)/빨강(위험)/회색(미확인) + "데이터 부재도 정보"라는 설계 |

### S06. 맥락 맞춤 대안 카드

> **"돌발 상황 시 대안 2~3개를 카드 형태로 즉시 제시. 검색 과정 자체가 사라진다."**

| 항목 | 내용 |
|------|------|
| 통합 아이디어 | L02(원탭대안카드), L06(3초대안카드), L13(대안카드3개), L18(스냅재조정) |
| 핵심 메커니즘 | 카드 3장 = 장소명+거리+리뷰+혼잡+추천이유 + 탭 한 번으로 선택 완료 |
| Needs Statement 연결 | "시간 매몰을 사전에 차단" = 검색 노동 23.5분→3분 이하 |

| 핵심 차별점 | 4명 독립 발상. 선택의 역설 제거. "열고, 보고, 탭한다" 3단계 |

### S07. 여행자 크라우드 리포팅 네트워크

> **"여행자가 현장 상태를 탭 2번으로 제보하면, 다음 여행자가 출발 전에 알림을 받는다."**

| 항목 | 내용 |
|------|------|
| 통합 아이디어 | B02(크라우드리포팅), L08(현장제보), C06(자동화봇) |
| 핵심 메커니즘 | 탭 2번 제보(4버튼) + Waze식 자정 메커니즘(맞아요/아니에요) + 포인트 보상 + API 데이터 보완 |
| Needs Statement 연결 | 정보 정확도 90% 달성의 핵심 경로. 구글맵 갱신율 30% 미만의 보완 수단 |
| B→A→C 커버리지 | **B의 전제 조건** (정보 정확도 = 서비스 존재의 이유) |
| 핵심 차별점 | 데이터 경쟁 방어벽(moat) 형성. 커뮤니티 기반 데이터 플라이휠 |
| 구현 난이도 | 중간 (기능 단순, 크리티컬 매스 확보가 진짜 난관) |

### S08. 감정 보호 단계적 대응 시스템

> **"상황 심각도에 따라 알림의 강도, 톤, 정보량을 단계적으로 조절하여 감정 절벽을 완만한 경사로 전환한다."**

| 항목 | 내용 |
|------|------|
| 통합 아이디어 | B09(감정방파제-인사이트), B11(감정방파제-아키), B27(에스컬레이션-파이프) |
| 핵심 메커니즘 | Level 1(경미→배너) / Level 2(주의→팝업+대안) / Level 3(위기→전체재조정) + 양방향 에스컬레이션 |
| Needs Statement 연결 | "감정 절벽을 사전에 차단" = 감정 곡선 급락(-2.0)을 완만 하강(-0.5)으로 전환 |
| B→A→C 커버리지 | **B(감정 보호)** + C(심각도에 따라 재조정 수준 결정) |
| 핵심 차별점 | S01(모니터링)의 "HOW"를 정의. 같은 정보도 전달 방식에 따라 감정 영향이 다름 |
| 구현 난이도 | 중간 (심각도 판단 로직 + UX 톤 설계) |

### S09. AI 현지 친구 대화형 컨시어지

> **"검색 엔진이 아니라 '그 동네 사는 한국인 친구'에게 톡 보내는 경험."**
